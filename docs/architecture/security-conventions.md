# 安全规范

> 本文档定义爱创作（AI Creation）项目的安全开发规范，涵盖认证、授权、敏感数据、接口安全与文件安全。

---

## 1. 认证与 JWT

### 1.1 JWT 配置

- 用户端与管理端使用 **独立的 Secret**，禁止共用。
- Secret 长度不少于 256 位，通过环境变量注入，禁止硬编码。
- Access Token 有效期 2 小时，Refresh Token 有效期 7 天。
- Token 在 `Authorization: Bearer {token}` 中传递。
- Token 失效后立即加入本地黑名单（Caffeine 缓存），有效期与 Token 剩余时间一致。

### 1.2 登录安全

- 密码错误次数限制：5 分钟内连续失败 5 次，锁定账号 30 分钟。
- 登录接口单独限流：同一 IP 每分钟不超过 10 次。
- 验证码有效期 5 分钟，使用一次后立即失效。

---

## 2. 密码与敏感数据

### 2.1 密码存储

- 用户密码使用 **BCrypt** 加密，强度因子 `10-12`。
- 禁止明文存储、禁止 MD5/SHA1。
- 管理端账号密码同样使用 BCrypt。

### 2.2 敏感字段脱敏

| 字段 | 存储 | 日志 | 接口返回 |
|---|---|---|---|
| 密码/哈希 | BCrypt 加密 | 不记录 | 不返回 |
| 手机号 | 明文 | 脱敏 `138****8000` | 脱敏 |
| 邮箱 | 明文 | 脱敏 `a***@example.com` | 脱敏 |
| JWT Token | 不存储 | 不记录完整 Token | 不返回他人 Token |
| LLM API Key | 加密存储 | 不记录 | 不返回 |
| 身份证号 | 加密存储 | 脱敏 | 脱敏 |

### 2.3 日志脱敏工具

提供统一脱敏工具 `DesensitizationUtil`，所有日志、异常信息、操作记录必须经过脱敏处理。

---

## 3. API Key 与 LLM 密钥

- OpenAI / 文心一言等 LLM API Key 通过环境变量或配置中心注入。
- 数据库中存储的 API Key 使用 AES 加密。
- 禁止将 API Key 打印到日志、返回给前端、提交到 Git。
- 调用 LLM 时设置超时和重试，防止密钥泄露后无限扣费。

---

## 4. 授权与权限

### 4.1 用户端权限

- 所有数据查询必须带 `user_id` 过滤，禁止越权访问。
- 用户只能操作自己的文章、任务、订单、风格预设。
- 管理端接口必须独立认证，普通用户 Token 无法访问。

### 4.2 管理端权限

- 基于 RBAC：管理员 → 角色 → 权限。
- 使用 Spring Security `@PreAuthorize("hasAuthority('admin:user:read')")` 控制接口。
- 超级管理员角色编码固定为 `SUPER_ADMIN`，不可删除。

### 4.3 权限编码规范

```text
{端}:{模块}:{动作}
```

示例：`admin:user:read`、`admin:user:write`、`admin:config:delete`

---

## 5. 接口安全

### 5.1 幂等与防重放

- 创建、支付、额度扣减接口必须幂等，通过 `Idempotency-Key` 实现。
- 幂等键有效期 24 小时，服务端记录已处理键。

### 5.2 限流

- 本地限流按用户 ID 或 IP 维度。
- 登录、注册、验证码接口单独限流。
- 触发限流返回 `429` + 错误码 `100003`。

### 5.3 CORS

- 后端统一配置 CORS，禁止 `*` 通配生产环境。
- 仅允许配置的域名访问。

### 5.4 CSRF

- 前后端分离项目，通过 JWT + CORS 白名单防护，不依赖 CSRF Token。

---

## 6. 文件上传安全

- 限制文件类型：仅允许图片（jpg/png/webp）和文档（doc/docx/pdf）。
- 限制文件大小：图片不超过 5MB，文档不超过 10MB。
- 文件保存路径不可预测，使用随机文件名。
- 禁止用户上传文件直接执行，禁止保存到 Web 根目录。
- 图片访问通过 Nginx 静态代理，不经过应用服务器。

---

## 7. 安全日志

- 登录、登出、密码修改、权限变更必须记录操作日志。
- 敏感操作（如导出用户数据、修改系统配置）需记录管理员 ID、IP、操作内容。
- 日志中不得包含明文密码、Token、API Key。

---

## 8. 禁止事项

- 禁止 SQL 拼接，必须使用参数化查询或 MyBatis-Plus Wrapper。
- 禁止返回完整堆栈给前端（生产环境）。
- 禁止在 URL 中传递敏感参数（如密码、Token）。
- 禁止信任前端传入的用户 ID，必须从 Token 中解析。
- 禁止将生产环境密钥、配置文件提交到 Git。
- 禁止跨端复用 JWT Secret 或认证接口。
