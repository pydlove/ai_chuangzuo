# 管理端基础

本域共 9 张表，覆盖管理员账号、RBAC 权限、AI 模型配置、会员到期提醒。

## a_admin_user — 管理员账号表

**用途**：存储管理端登录账号，内置超管 `admin`。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| username | VARCHAR(64) | 否 | - | 登录账号 |
| password_hash | VARCHAR(256) | 否 | - | 密码哈希（BCrypt） |
| real_name | VARCHAR(64) | 是 | NULL | 真实姓名 |
| phone | VARCHAR(20) | 是 | NULL | 手机号 |
| email | VARCHAR(128) | 是 | NULL | 邮箱 |
| avatar_url | VARCHAR(512) | 是 | NULL | 头像URL |
| status | TINYINT UNSIGNED | 否 | 1 | 状态：0-禁用，1-启用 |
| last_login_at | DATETIME(3) | 是 | NULL | 最后登录时间 |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 是否删除：0-否，1-是（逻辑删除） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人ID |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_a_admin_user_username` (`username`)

**说明**：通过 `a_admin_user_role_rel` 关联到 `a_role`，一个管理员可拥有多个角色。`is_deleted` 为 MyBatis-Plus 逻辑删除标记。

## a_admin_login_log — 管理员登录日志表

**用途**：记录管理员登录成功/失败流水，用于安全审计。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| admin_user_id | BIGINT UNSIGNED | 否 | 0 | 管理员ID，0 表示未登录（如账号不存在时的失败尝试） |
| login_type | TINYINT UNSIGNED | 否 | 1 | 登录类型：1-密码登录 |
| client_ip | VARCHAR(45) | 是 | NULL | 客户端IP（兼容 IPv6） |
| user_agent | VARCHAR(512) | 是 | NULL | User-Agent |
| login_status | TINYINT UNSIGNED | 否 | 1 | 登录状态：0-失败，1-成功 |
| fail_reason | VARCHAR(256) | 是 | NULL | 失败原因（成功时为 NULL） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |

**索引**：
- PRIMARY KEY (`id`)
- KEY `idx_a_admin_login_log_admin_user_id` (`admin_user_id`)
- KEY `idx_a_admin_login_log_created_at` (`created_at`)

**说明**：仅追加，不更新；按 `admin_user_id` + `created_at` 检索某管理员的登录历史。

## a_role — 角色表

**用途**：RBAC 中的角色定义，内置 `SUPER_ADMIN` 超级管理员角色。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| role_code | VARCHAR(64) | 否 | - | 角色编码（如 SUPER_ADMIN） |
| role_name | VARCHAR(64) | 否 | - | 角色名称 |
| description | VARCHAR(256) | 是 | NULL | 描述 |
| status | TINYINT UNSIGNED | 否 | 1 | 状态：0-禁用，1-启用 |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 是否删除：0-否，1-是（逻辑删除） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_a_role_code` (`role_code`)

**说明**：通过 `a_admin_user_role_rel` 关联管理员，通过 `a_role_permission_rel` 关联权限。

## a_permission — 权限表

**用途**：RBAC 中的权限点定义，支持菜单/按钮/接口三种资源，可树形嵌套。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| permission_code | VARCHAR(128) | 否 | - | 权限编码（如 `system:user:list`） |
| permission_name | VARCHAR(128) | 否 | - | 权限名称 |
| resource_type | TINYINT UNSIGNED | 否 | 1 | 资源类型：1-菜单，2-按钮，3-接口 |
| parent_id | BIGINT UNSIGNED | 否 | 0 | 父权限ID，0 表示顶级 |
| sort_order | INT UNSIGNED | 否 | 0 | 同级排序，越小越靠前 |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 是否删除：0-否，1-是（逻辑删除） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_a_permission_code` (`permission_code`)

**说明**：通过 `parent_id` 自关联形成权限树；通过 `a_role_permission_rel` 关联角色。

## a_admin_user_role_rel — 管理员角色关联表

**用途**：管理员与角色的多对多关联。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| admin_user_id | BIGINT UNSIGNED | 否 | - | 管理员ID（→ a_admin_user.id） |
| role_id | BIGINT UNSIGNED | 否 | - | 角色ID（→ a_role.id） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_a_admin_user_role_rel` (`admin_user_id`, `role_id`)

**说明**：RBAC 用户↔角色桥表，唯一键防止重复授权。

## a_role_permission_rel — 角色权限关联表

**用途**：角色与权限的多对多关联。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| role_id | BIGINT UNSIGNED | 否 | - | 角色ID（→ a_role.id） |
| permission_id | BIGINT UNSIGNED | 否 | - | 权限ID（→ a_permission.id） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_a_role_permission_rel` (`role_id`, `permission_id`)

**说明**：RBAC 角色↔权限桥表，唯一键防止重复授权。

## a_model_config — AI 模型配置表

**用途**：管理端配置 AI 厂商接入参数，每个厂商一行，全局仅一条 `is_active=1`。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| provider_type | VARCHAR(64) | 否 | - | 厂商类型：kimi / minimax |
| base_url | VARCHAR(512) | 否 | - | API 基础地址 |
| api_key_encrypted | VARCHAR(512) | 否 | - | 加密后的 API Key（AES 加密存储） |
| model_code | VARCHAR(128) | 否 | - | 模型编码（如 moonshot-v1-8k） |
| model_name | VARCHAR(128) | 是 | NULL | 模型显示名 |
| is_active | TINYINT UNSIGNED | 否 | 0 | 是否启用：0-否，1-是（全局唯一） |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 是否删除：0-否，1-是（逻辑删除） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人ID |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_a_model_config_provider_type` (`provider_type`)

**说明**：`provider_type` 唯一约束保证每个厂商仅一条配置；启用新配置时由业务层把其它行 `is_active` 置 0，保证全局只有一条启用。

## reminder_config — 到期提醒配置表

**用途**：会员到期提醒的全局单行配置，固定 `id=1`。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | - | 主键ID，固定为 1 |
| advance_days | INT | 否 | 7 | 提前提醒天数 N（命中区间：0 ≤ remainingDays ≤ N） |
| notify_hour | TINYINT UNSIGNED | 否 | 9 | 每天提醒时间点，取值 0-23 |
| notify_channel | VARCHAR(16) | 否 | 'message' | 通知形式：message-站内信 / email-邮件 / message_email-站内信+邮件 |
| enabled | TINYINT UNSIGNED | 否 | 1 | 定时提醒开关：0-关，1-开 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人ID |

**索引**：
- PRIMARY KEY (`id`)

**说明**：单行配置表，初始化时 `INSERT IGNORE` 写入 id=1 的默认行；运行时只 UPDATE 不 INSERT。

## u_reminder_send_log — 到期提醒发送记录表

**用途**：记录每次到期提醒的发送流水，按 (user_id, channel, send_date) 去重。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| user_id | BIGINT UNSIGNED | 否 | - | 被提醒的用户ID |
| channel | VARCHAR(16) | 否 | - | 发送渠道：message-站内信 / email-邮件 |
| send_date | DATE | 否 | - | 发送日期（本地日） |
| remaining_days | INT | 否 | - | 发送时剩余天数 |
| trigger_type | VARCHAR(8) | 否 | - | 触发类型：auto-定时自动 / manual-手动触发 |
| status | TINYINT UNSIGNED | 否 | 1 | 发送状态：0-失败，1-成功 |
| fail_reason | VARCHAR(256) | 是 | NULL | 失败原因（成功时为 NULL） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |

**索引**：
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_reminder_user_channel_date` (`user_id`, `channel`, `send_date`)
- KEY `idx_reminder_send_date` (`send_date`)

**说明**：唯一键保证同一用户同一渠道同一天只成功发送一次；`trigger_type=manual` 时同样受唯一键约束。
