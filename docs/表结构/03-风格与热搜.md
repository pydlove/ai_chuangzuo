# 风格与热搜

本域共 5 张表，覆盖用户写作风格、风格市场、热搜平台榜单与抓取配置。

## u_user_style — 用户写作风格表

**用途**：存储用户的写作风格配置，包括自定义风格、学习他人风格、系统预设风格。生成文章时作为提示词注入。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| biz_no | VARCHAR(64) | 否 | - | 业务唯一编号，对外暴露 |
| user_id | BIGINT UNSIGNED | 否 | - | 所属用户ID；系统预设风格为 0 |
| style_name | VARCHAR(64) | 否 | - | 风格名称；同一用户下唯一 |
| description | VARCHAR(256) | 是 | NULL | 简短描述（系统预选用） |
| prompt_summary | VARCHAR(512) | 是 | NULL | 提示词摘要，UI 卡片展示用（系统预选用） |
| prompt | TEXT | 否 | - | 风格提示词，写入生成请求 |
| scope | VARCHAR(256) | 是 | NULL | 适用范围标签，逗号分隔 |
| enable_status | TINYINT UNSIGNED | 否 | 1 | 启用状态：0-禁用，1-启用（仅系统预设有意义） |
| source_type | TINYINT UNSIGNED | 否 | 1 | 来源类型：1-自定义，2-学习，3-系统预设 |
| use_count | INT UNSIGNED | 否 | 0 | 累计使用次数 |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删，1-已删 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| audit_status | TINYINT UNSIGNED | 否 | 1 | 审核状态：0-待审核，1-已通过，2-已拒绝 |
| audited_by | BIGINT UNSIGNED | 是 | NULL | 审核管理员ID |
| audited_at | DATETIME(3) | 是 | NULL | 审核时间 |
| reject_reason | VARCHAR(256) | 是 | NULL | 打回原因 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_user_style_biz_no (biz_no)
- UNIQUE KEY uk_u_user_style_user_id_name (user_id, style_name)
- KEY idx_u_user_style_user_id_source (user_id, source_type)
- KEY idx_u_user_style_audit_status (audit_status)

**说明**：
- `source_type` 取值：1-自定义，2-学习，3-系统预设（V1.0.0_013 扩展）。
- `audit_status` 取值：0-待审核，1-已通过，2-已拒绝（见 `AuditStatus` 枚举）。
- 系统预设风格 `user_id = 0`，`source_type = 3`，初始 seed 8 条（年度总结、产品评测、情感散文、职场干货、热点评论、知识科普、营销转化、故事叙事）。
- 管理端通过 `UserStyleAggregate` 读模型访问本表；表中无 `created_by` / `updated_by` 列。

## u_style_market — 风格市场表

**用途**：存储上架到风格市场的风格商品，用户可购买/使用他人发布的风格，平台运营也可直接发布。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键ID |
| biz_no | VARCHAR(64) | 否 | - | 业务唯一编号，对外暴露 |
| style_name | VARCHAR(64) | 否 | - | 风格名称 |
| description | VARCHAR(256) | 是 | NULL | 简短描述 |
| prompt_summary | VARCHAR(512) | 是 | NULL | 提示词摘要，UI 卡片展示用 |
| prompt | TEXT | 否 | - | 完整风格提示词 |
| scope | VARCHAR(256) | 是 | NULL | 适用范围标签，逗号分隔 |
| publisher_user_id | BIGINT UNSIGNED | 否 | - | 发布者用户ID（u_user.id） |
| price | DECIMAL(10,2) | 否 | 0.20 | 单次使用价格（创作币），本期固定 0.2 |
| total_uses | INT UNSIGNED | 否 | 0 | 累计使用次数（管理端可维护） |
| weekly_uses | INT UNSIGNED | 否 | 0 | 本周使用次数（结算用，本期只读） |
| weekly_earnings | DECIMAL(10,2) | 否 | 0.00 | 本周基础收益（本期只读） |
| milestone_bonus | DECIMAL(10,2) | 否 | 0.00 | 本周已发放里程碑奖励（本期只读） |
| last_settlement_at | DATETIME(3) | 是 | NULL | 上次结算时间（本期只读） |
| enable_status | TINYINT UNSIGNED | 否 | 1 | 启用状态：0-禁用，1-启用 |
| audit_status | TINYINT UNSIGNED | 否 | 1 | 审核状态：0-待审核，1-已通过，2-已拒绝 |
| source_type | TINYINT UNSIGNED | 否 | 3 | 来源：1-用户自定义，2-学习风格，3-平台运营 |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删，1-已删 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_style_market_biz_no (biz_no)
- KEY idx_style_market_publisher (publisher_user_id)
- KEY idx_style_market_enable_deleted (enable_status, is_deleted)
- KEY idx_style_market_audit (audit_status)
- KEY idx_style_market_name (style_name)

**说明**：
- `audit_status` 取值：0-待审核，1-已通过，2-已拒绝（见 `AuditStatus` 枚举）。
- `source_type` 取值：1-用户自定义，2-学习风格，3-平台运营。
- `publisher_user_id` 关联 `u_user.id`。
- 用户端与管理端各有一个 `StyleMarket` 实体映射本表，字段一致。

## hot_search_platform — 热搜平台配置表

**用途**：存储支持抓取热搜的平台列表，如抖音、今日头条、B站、微博、百度。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT | 否 | AUTO_INCREMENT | 主键ID |
| code | VARCHAR(32) | 否 | - | 平台编码：douyin、toutiao、bilibili、weibo、baidu |
| name | VARCHAR(64) | 否 | - | 平台名称 |
| icon | VARCHAR(255) | 是 | NULL | 平台图标 URL |
| sort_order | INT | 否 | 0 | 展示排序 |
| enabled | TINYINT | 否 | 1 | 是否启用：0-否，1-是 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | 否 | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_hot_search_platform_code (code)

**说明**：
- 初始 seed 5 个平台：douyin（抖音）、toutiao（今日头条）、bilibili（B 站）、weibo（微博）、baidu（百度）。
- `hot_search_daily.platform_code` 关联本表 `code`。

## hot_search_daily — 每日热搜榜单表

**用途**：存储每日各平台热搜榜单快照，用于创作时参考热点。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT | 否 | AUTO_INCREMENT | 主键ID |
| platform_code | VARCHAR(32) | 否 | - | 平台编码 |
| rank_num | INT | 否 | - | 排名 |
| title | VARCHAR(512) | 否 | - | 热搜标题 |
| hot_value | VARCHAR(64) | 是 | NULL | 热度值字符串 |
| url | VARCHAR(1024) | 是 | NULL | 跳转链接 |
| search_count | BIGINT | 是 | NULL | 搜索量数字 |
| snapshot_date | DATE | 否 | - | 快照日期 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_hot_search_daily_platform_date_rank (platform_code, snapshot_date, rank_num)

**说明**：
- `platform_code` 关联 `hot_search_platform.code`。
- 通过 `(platform_code, snapshot_date, rank_num)` 唯一约束防止同一平台同一天同一排名重复入库。

## hot_search_config — 热搜抓取配置表

**用途**：单行配置表，存储热搜定时抓取的调度与超时参数。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | - | 主键ID，固定为 1 |
| cron | VARCHAR(64) | 否 | - | 抓取任务 cron 表达式 |
| enabled | TINYINT UNSIGNED | 否 | 1 | 是否启用：0-否，1-是 |
| top_n | INT | 否 | 50 | 每次抓取保留前 N 条 |
| connect_timeout_millis | INT | 否 | 5000 | 连接超时时间（毫秒） |
| read_timeout_millis | INT | 否 | 10000 | 读取超时时间（毫秒） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人ID |

**索引**：
- PRIMARY KEY (id)

**说明**：
- 单行表，固定 `id = 1`，初始配置：`cron = '0 0 2 * * ?'`（每天凌晨 2 点），`enabled = 1`，`top_n = 50`。
- 仅管理端有对应实体 `HotSearchConfig`，用户端无此表实体。
