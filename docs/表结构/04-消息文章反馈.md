# 消息、文章与反馈

本域共 5 张表，覆盖站内消息、用户已读、文章作品、草稿、用户反馈。

## u_message — 站内消息表

**用途**：存储站内消息内容。广播消息（scope=1）单行落库，所有用户共享；个人消息（scope=2）每个目标用户一行。管理端一次发布指定人消息会扇出多行，通过 `biz_no` 聚合。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| biz_no | VARCHAR(32) | 否 | '' | 批次号：广播单行独立；指定人一次发布的多行共享同一 biz_no；历史数据回填为 `LEGACY{id}` |
| msg_type | VARCHAR(32) | 否 | — | 消息类型：announcement-公告 / feature-新功能 / promotion-优惠活动 / generation-生成完成 / membership-会员提醒 |
| scope | TINYINT UNSIGNED | 否 | — | 范围：1-广播（全体），2-个人 |
| target_user_id | BIGINT UNSIGNED | 是 | NULL | 目标用户 ID：个人消息填写，广播为 NULL |
| title | VARCHAR(128) | 否 | — | 标题 |
| summary | VARCHAR(512) | 否 | — | 摘要（列表展示用） |
| content | MEDIUMTEXT | 是 | NULL | 完整正文，前端详情弹框用；summary 是列表摘要 |
| sub_type | VARCHAR(32) | 是 | NULL | 子类型，如 membership.subscribed / membership.expiring / feedback.reply |
| link_url | VARCHAR(256) | 是 | NULL | 点击跳转路由，空则前端按类型默认跳转 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- KEY idx_message_scope_type (scope, msg_type)
- KEY idx_message_target (target_user_id)
- KEY idx_message_created (created_at)
- KEY idx_message_subtype (sub_type)
- KEY idx_message_biz_no (biz_no)
- KEY idx_message_msg_type_biz_no (msg_type, biz_no)

**说明**：
- `scope` 取值见 `MessageScope` 枚举：1=BROADCAST 广播、2=PERSONAL 个人。
- 个人消息通过 `target_user_id` 关联 `u_user.id`；用户的已读状态在 `u_message_read` 中记录。
- 管理端 `MessageAggregate` 与 `reminder.Message` 均映射本表：前者用于按 `biz_no` 聚合查询，后者仅 insert（如会员到期提醒，bizNo=RMD-uuid16）。

## u_message_read — 用户消息已读记录表

**用途**：记录用户对消息的已读行为。广播消息每个用户读完写一行；个人消息读完也写一行。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| user_id | BIGINT UNSIGNED | 否 | — | 用户 ID |
| message_id | BIGINT UNSIGNED | 否 | — | 消息 ID |
| read_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 已读时间 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_message_read_user_msg (user_id, message_id)
- KEY idx_message_read_message (message_id)

**说明**：
- `user_id` → `u_user.id`，`message_id` → `u_message.id`。
- 唯一约束 `uk_message_read_user_msg` 保证同一用户对同一消息只产生一条已读记录，幂等。
- 管理端统计某批次（biz_no）消息的已读率时，按 `u_message.id IN (...)` JOIN 本表。

## u_article — 用户已生成作品表

**用途**：存储用户 AI 生成并已完成的文章作品。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| biz_no | VARCHAR(64) | 否 | — | 业务唯一编号，对外暴露 |
| user_id | BIGINT UNSIGNED | 否 | — | 所属用户 ID（u_user.id） |
| title | VARCHAR(256) | 否 | — | 作品标题 |
| body | MEDIUMTEXT | 否 | — | 作品正文 |
| style_overrides | JSON | 是 | NULL | 编辑器内联样式覆盖，用于 Preview/Edit 还原；形如 `{"blocks":{},"inlines":[{"block":0,"start":1,"end":5,"styles":{...}}]}` |
| platform | VARCHAR(32) | 是 | NULL | 发布平台 key：wechat / xiaohongshu / toutiao / baijiahao / douyin / zhihu / general |
| style | VARCHAR(64) | 是 | NULL | 写作风格名称 |
| template | VARCHAR(64) | 是 | NULL | 导出模板名称 |
| word_count | INT UNSIGNED | 否 | 0 | 正文字数（去空白字符） |
| completed_at | DATETIME(3) | 是 | NULL | 生成完成时间 |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_article_biz_no (biz_no)
- KEY idx_u_article_user_deleted (user_id, is_deleted)
- KEY idx_u_article_completed (user_id, completed_at)

**说明**：
- `user_id` → `u_user.id`。
- 与 `u_draft` 的区别：本表是已生成完成的作品，`u_draft` 是创作中的草稿（未提交生成或生成前保存）。
- 注意：V2.0.0_029 的 `cover_image_url` 加在 `t_article`（创作学院文章），与本表无关。

## u_draft — 用户创作草稿表

**用途**：存储用户在创作页保存的草稿（标题、需求描述、平台、字数等输入参数），尚未生成正文。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| biz_no | VARCHAR(64) | 否 | — | 业务唯一编号，对外暴露 |
| user_id | BIGINT UNSIGNED | 否 | — | 所属用户 ID（u_user.id） |
| custom_title | VARCHAR(256) | 是 | NULL | 草稿标题（用户自定义） |
| custom_requirement | TEXT | 是 | NULL | 草稿需求描述（用户自定义） |
| platform | VARCHAR(32) | 是 | NULL | 发布平台 key |
| word_count | INT UNSIGNED | 否 | 0 | 目标字数 |
| style | VARCHAR(64) | 是 | NULL | 写作风格名称 |
| template | VARCHAR(64) | 是 | NULL | 导出模板名称 |
| saved_at | DATETIME(3) | 否 | — | 草稿保存时间（用户最近一次编辑的时间） |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_draft_biz_no (biz_no)
- KEY idx_u_draft_user_deleted (user_id, is_deleted)
- KEY idx_u_draft_saved (user_id, saved_at)

**说明**：
- `user_id` → `u_user.id`。
- 草稿提交生成并完成后，正文落入 `u_article`，本表数据可保留或按业务清理。

## u_feedback — 用户反馈表

**用途**：存储用户提交的反馈，以及管理员的回复。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| user_id | BIGINT UNSIGNED | 否 | — | 提交人用户 ID |
| type | VARCHAR(32) | 否 | — | 反馈类型 |
| content | VARCHAR(2000) | 否 | — | 反馈正文 |
| reply_content | VARCHAR(2000) | 是 | NULL | 管理员回复内容 |
| reply_admin_id | BIGINT UNSIGNED | 是 | NULL | 回复管理员 ID |
| replied_at | DATETIME(3) | 是 | NULL | 回复时间 |
| status | TINYINT UNSIGNED | 否 | 0 | 状态：0-待回复，1-已回复 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- KEY idx_u_feedback_user_created (user_id, created_at)
- KEY idx_u_feedback_status_created (status, created_at)

**说明**：
- `user_id` → `u_user.id`；`reply_admin_id` → `a_admin.id`。
- 历史曾有 `contact` 字段，V1.0.0_022 已删除：用户已用邮箱注册，admin 通过 `u_user.email` 联系，无需重复填写联系方式。
- 管理端通过 `AdminFeedbackView` 联表查询，附带 `userEmail`、`userBizNo`、`replyAdminName` 等展示字段。
