# AI 生成

本域共 7 张表，覆盖生成任务、生成历史、生成配置、调用日志、提示词模板（含阶段、版本）。

## a_generation_task — 生成任务表

**用途**：文章生成队列表。user-api 提交任务，admin-api worker 按 FIFO 拉取并跑 12 阶段流水线；任务终态后由 retention job 归档到 `a_generation_history`。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 |
| biz_no | VARCHAR(64) | 否 | — | 业务唯一编号（对外暴露，如 GA20260709xxxx） |
| target_user_id | BIGINT UNSIGNED | 否 | — | 发起任务的用户 ID |
| status | TINYINT UNSIGNED | 否 | 0 | 状态：0-queued，1-processing，2-completed，3-failed |
| model_config_id | BIGINT UNSIGNED | 否 | — | 使用的 AI 模型配置 ID |
| prompt_template_id | BIGINT UNSIGNED | 否 | — | 使用的提示词模板 ID（提交时快照） |
| prompt_template_version | INT UNSIGNED | 是 | NULL | 任务创建时锁定的模板版本号；NULL=fallback 到 enabled=1 的最新版 |
| input_param | JSON | 否 | — | 输入参数：title/description/platform/wordCount/styleRef/toneTags |
| word_limit_target | INT UNSIGNED | 否 | 3000 | 用户要求字数（≤3000） |
| progress_pct | TINYINT UNSIGNED | 否 | 0 | 任务进度 0-100；worker 每阶段结束后写回；user 端轮询可见 |
| retry_count | INT UNSIGNED | 否 | 0 | 已重试次数 |
| max_retry | INT UNSIGNED | 否 | 3 | 最大重试次数 |
| locked_at | DATETIME(3) | 是 | NULL | worker 锁定（开始处理）时间 |
| locked_by | VARCHAR(64) | 是 | NULL | worker 实例 ID，当前单实例固定 worker-1 |
| lease_until | DATETIME(3) | 是 | NULL | lease 超时时刻（locked_at + 5 分钟） |
| failed_reason | VARCHAR(512) | 是 | NULL | 最后一次失败原因 |
| completed_at | DATETIME(3) | 是 | NULL | 完成时间（成功或最终失败） |
| retention_days | INT | 是 | NULL | 保留天数：基础 30，pro/旗舰 NULL=永久 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID（=0） |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY `uk_a_gt_biz_no` (biz_no)
- KEY `idx_a_gt_user_created` (target_user_id, created_at) — FIFO 拉取 + 用户隔离
- KEY `idx_a_gt_status_lease` (status, lease_until) — worker 扫候选任务 / lease 回收

**说明**：worker 通过 `status + lease_until` 抢占任务并写入 `locked_at/locked_by/lease_until`；lease 超时后任务可被其他 worker 回收。任务终态（2/3）后由 retention job 按 `retention_days` 归档到 `a_generation_history`，本表只保留活跃 + 未到期任务。

## a_generation_history — 生成历史归档表

**用途**：生成任务冷存储。retention job 把过期的 `a_generation_task` 终态任务迁到此表，字段做了裁剪（去掉 updated_at/updated_by/is_deleted/retention_days/locked_by 等运行时字段）。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 归档表自增主键 |
| task_id | BIGINT UNSIGNED | 否 | — | 原 a_generation_task.id |
| biz_no | VARCHAR(64) | 否 | — | 业务唯一编号 |
| target_user_id | BIGINT UNSIGNED | 否 | — | 发起任务的用户 ID |
| title | VARCHAR(256) | 是 | NULL | 任务结束时已确定的 title（成功或失败状态都有） |
| input_param | JSON | 否 | — | 输入参数 JSON |
| status | TINYINT UNSIGNED | 否 | — | 终态：2-completed，3-failed |
| retry_count | INT UNSIGNED | 否 | 0 | 累计重试次数 |
| failed_reason | VARCHAR(512) | 是 | NULL | 失败原因（status=failed 时） |
| word_limit_target | INT UNSIGNED | 否 | — | 用户要求字数 |
| model_config_id | BIGINT UNSIGNED | 否 | — | 使用的 AI 模型配置 ID |
| prompt_template_id | BIGINT UNSIGNED | 否 | — | 使用的提示词模板 ID |
| created_at | DATETIME(3) | 否 | — | 原任务创建时间 |
| completed_at | DATETIME(3) | 是 | NULL | 完成时间 |
| duration_ms | BIGINT | 是 | NULL | 处理耗时（毫秒） |
| archived_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 入档时间 |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY `uk_a_gh_task_id` (task_id)
- KEY `idx_a_gh_user_archived` (target_user_id, archived_at)

**说明**：与 `a_generation_task` 是 1:1 归档关系（`task_id` 唯一）。只存终态任务，用于历史查询、对账、避免主表膨胀。

## a_generation_config — 生成运行时配置表

**用途**：创作运行时单行配置（id=1）。由 admin 端 GET/PUT 改写；worker 启动时读 + 每 10s 刷新。`pool_size` 变更需重启 admin-api 生效，其他字段下一轮轮询即生效。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键（固定 1） |
| pool_size | INT UNSIGNED | 否 | 2 | worker 线程池大小（1-10，重启生效） |
| claim_batch_size | INT UNSIGNED | 否 | 1 | 每轮单次拉取任务数（1-10） |
| lease_minutes | INT UNSIGNED | 否 | 5 | lease 持续分钟（1-60） |
| max_retry | INT UNSIGNED | 否 | 3 | 单任务最大重试次数（1-10） |
| poll_interval_ms | INT UNSIGNED | 否 | 500 | 空轮询间隔 ms（100-5000） |
| retention_cron | VARCHAR(64) | 否 | '0 0 3 * * ?' | 归档定时任务 cron 表达式 |
| worker_id | VARCHAR(64) | 否 | 'worker-1' | worker 实例 ID（单实例约定） |
| llm_retry_max_attempts | INT UNSIGNED | 否 | 3 | 单 stage AI 调用最大尝试次数（含首次；1-10） |
| llm_retry_base_delay_ms | INT UNSIGNED | 否 | 500 | 首次重试前等待 ms（100-10000） |
| llm_retry_backoff_multiplier | INT UNSIGNED | 否 | 2 | 指数退避倍数（1-5：第 N 次重试睡 baseDelay * multiplier^(N-1)） |
| default_temperature | DECIMAL(3,2) | 否 | 0.70 | AI temperature 默认值（0.00-2.00；stage model_params 可覆盖） |
| default_max_tokens | INT UNSIGNED | 否 | 32768 | AI max_tokens 默认值（1-128000；MiniMax-M3 等推理模型 reasoning 也吃此预算，润色类 stage 至少给到 32k） |
| default_top_p | DECIMAL(3,2) | 否 | 1.00 | AI top_p 默认值（0.00-1.00；stage model_params 可覆盖） |
| ai_read_timeout_seconds | INT UNSIGNED | 否 | 180 | AI 调用读取超时（秒，30-600） |
| remark | VARCHAR(256) | 是 | NULL | 备注 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID（=0） |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)

**说明**：单行配置表。stage 级 `model_params` 优先于本表 `default_*`；未配置时回落到这里的全局默认。`default_max_tokens` 历经 8192 → 16384 → 32768 三次调大，原因是 MiniMax-M3 推理模型的 reasoning 与 content 共享 token 预算。

## a_generation_call_log — 生成 AI 调用日志表

**用途**：记录 worker 跑 12 阶段流水线时每次 AI 调用的明细（stage / 尝试次数 / 成功 / 错误 / 耗时 / 完整输入输出）。任务结束（成功或失败）后批量落库；crash 中途丢失记录。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 |
| task_id | BIGINT UNSIGNED | 否 | — | 所属任务 ID（a_generation_task.id） |
| stage_index | INT UNSIGNED | 否 | — | 阶段序号 1-12 |
| stage_name | VARCHAR(32) | 否 | — | 阶段稳定标识符：outline / draft / ... |
| attempt | INT UNSIGNED | 否 | 1 | 第几次尝试（1=首次，2=第 1 次重试，...） |
| success | TINYINT UNSIGNED | 否 | 1 | 是否成功：0-失败，1-成功 |
| error | VARCHAR(512) | 是 | NULL | 失败原因（成功时为 NULL） |
| user_msg | TEXT | 是 | NULL | 本次尝试完整 userMsg（变量已替换） |
| response_content | TEXT | 是 | NULL | AI 完整返回（成功时） |
| duration_ms | INT UNSIGNED | 否 | 0 | 本次尝试耗时（ms） |
| called_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 调用开始时间 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID（=0） |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |

**索引**：
- PRIMARY KEY (id)
- KEY `idx_log_task` (task_id, called_at)
- KEY `idx_log_stage` (task_id, stage_index, attempt)

**说明**：与 `a_generation_task` 是 N:1 关系，一个任务对应多条调用日志（12 阶段 × 每阶段 1~N 次重试）。V2.0.0_031 把原来的 200 字 preview 列升级为 TEXT 全文列，便于排查「这任务为什么失败 / AI 完整返回了什么」。

## t_prompt_template — 创作提示词模板表

**用途**：创作提示词模板主表。多模板共存，但 runtime 仅 1 个 `template_status=1`（已发布）生效（V2.0.0_024 收敛为单发布，去掉 enabled 双轨字段）。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 |
| name | VARCHAR(64) | 否 | — | 模板名称（管理后台显示） |
| base_content | MEDIUMTEXT | 是 | NULL | 基础内容（去 AI 味） |
| template_status | TINYINT UNSIGNED | 否 | 0 | 模板状态：0-草稿，1-已发布，2-已下线 |
| latest_published_version | INT UNSIGNED | 是 | NULL | 当前最新已发布版本号，未发布则为 NULL |
| remark | VARCHAR(256) | 是 | NULL | 备注 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)

**说明**：
- 历史迁移：V2.0.0_011 初建含 `user_style_guidance` / `system_prompt_json` / `enabled` 列；V2.0.0_012 删除 `user_style_guidance`（改由 `u_user_style.prompt` 在任务提交时快照到 `input_param`）和 `system_prompt_json`（改为 Java 端 `PromptConstants.SYSTEM_PROMPT_JSON` 硬编码）；V2.0.0_018 加 `template_status` + `latest_published_version`；V2.0.0_024 删 `enabled`，发布语义只保留 `template_status`，并强制全表唯一 PUBLISHED。
- 与 `t_prompt_template_stage` 是 1:N 关系（一个模板对应 12 行 stage）；与 `t_prompt_template_version` 是 1:N 关系（每次发布落一份版本快照）。
- 内置默认模板 id=1（`默认去 AI 味模板`），由 V2.0.0_017 种子插入。

## t_prompt_template_stage — 提示词模板 12 阶段配置表

**用途**：一个模板对应 12 行 stage（stage_index 1-12），每行存对应阶段的 AI prompt 或规则配置。对应设计文档 `2026-07-09-de-ai-flavor-writing-pipeline-design.md`。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 |
| template_id | BIGINT UNSIGNED | 否 | — | 所属模板 ID（t_prompt_template.id） |
| stage_index | TINYINT UNSIGNED | 否 | — | 阶段序号 1-12（设计文档固定 12 阶段） |
| stage_type | VARCHAR(16) | 否 | — | 阶段类型：ai_prompt / rule_config / passthrough |
| stage_key | VARCHAR(32) | 否 | — | 阶段稳定标识符：outline / material_list / draft / rhythm_detect / ... |
| ai_prompt | MEDIUMTEXT | 是 | NULL | 仅 stage_type=ai_prompt 有值：可编辑的 AI 提示词模板 |
| rule_config | JSON | 是 | NULL | 仅 stage_type=rule_config 有值：规则参数（JSON） |
| model_params | JSON | 是 | NULL | AI 阶段可配参数：temperature / max_tokens / top_p 等；NULL=用全局默认 |
| enabled | TINYINT UNSIGNED | 否 | 1 | 该 stage 是否启用（admin 可单 stage 关停） |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID（=0） |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY `uk_template_stage` (template_id, stage_index)
- KEY `idx_stage_template` (template_id)

**说明**：
- `stage_type` 决定哪一列有值：`ai_prompt` → `ai_prompt` + 可选 `model_params`；`rule_config` → `rule_config`；`passthrough` → 全空。
- `model_params` 优先级高于 `a_generation_config.default_*`，未配置时回落到全局默认。
- 老模板（V2.0.0_011 创建）不会自动建 stage 行，admin 在编辑页点「初始化 12 阶段」才补齐；DB 未命中时 `PipelineTemplateResolver` 回落到 `PipelineStage` enum 默认值兜底。

## t_prompt_template_version — 提示词模板版本快照表

**用途**：模板每次发布时把当前 12 阶段配置完整快照成一行版本，便于回溯与生成任务锁定版本。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 |
| template_id | BIGINT UNSIGNED | 否 | — | 所属模板 ID |
| version | INT UNSIGNED | 否 | — | 版本号，从 1 开始自增 |
| version_status | TINYINT UNSIGNED | 否 | 0 | 版本状态：0-草稿，1-已发布，2-已下线 |
| config_json | JSON | 否 | — | 12 阶段配置完整快照（stages 数组，含 index/stageKey/stageType/aiPrompt/ruleConfig/enabled） |
| change_note | VARCHAR(512) | 是 | NULL | 本次发布变更说明 |
| published_at | DATETIME(3) | 是 | NULL | 发布时间 |
| published_by | BIGINT UNSIGNED | 是 | NULL | 发布人 ID |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY `uk_t_pt_version` (template_id, version)
- KEY `idx_t_pt_version_status` (version_status)
- KEY `idx_t_pt_version_template` (template_id)

**说明**：
- 与 `t_prompt_template` 是 N:1 关系，与 `t_prompt_template_stage` 是快照关系（发布时把当前 12 行 stage 序列化成 JSON 存到 `config_json`）。
- `a_generation_task.prompt_template_version` 锁定本表的 `version`，运行时 worker 按 (template_id, version) 读快照执行；若任务行的 version 为 NULL，fallback 到模板的 `latest_published_version`。
- V2.0.0_024 收敛单发布时，被下线模板的已发布版本行同步置为 `version_status=2`。
