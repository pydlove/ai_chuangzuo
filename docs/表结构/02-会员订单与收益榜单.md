# 会员订单与收益榜单

本域共 6 张表，覆盖会员订阅、订单支付、创作币流水、收益记录、榜单收入申报与奖励发放。

## u_user_membership — 用户会员状态表

**用途**：记录每个用户当前的会员等级与有效期，一个用户仅一行，随续费/升级被覆盖更新。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| user_id | BIGINT UNSIGNED | 否 | - | 用户 ID，关联 u_user.id |
| level | VARCHAR(32) | 否 | - | 当前会员等级：basic / pro / flagship |
| started_at | DATE | 否 | - | 本次会员开始日期 |
| expires_at | DATE | 否 | - | 会员到期日期 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_user_membership_user_id (user_id) — 一个用户仅一条会员记录
- KEY idx_u_user_membership_expires_at (expires_at) — 便于到期提醒/批量过期处理

**说明**：
- `user_id` 关联 `u_user.id`；`level` 与 `u_order.plan_key` 取值一致。
- 订单支付成功后由后端写入/覆盖本表；到期任务根据 `expires_at` 扫描降级。

## u_order — 用户订单表

**用途**：记录用户购买会员套餐的订单，包含待支付与已支付两种状态。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| order_no | VARCHAR(32) | 否 | - | 订单编号，格式 SUB{yyMMdd}{6位随机} |
| user_id | BIGINT UNSIGNED | 否 | - | 下单用户 ID，关联 u_user.id |
| plan_key | VARCHAR(32) | 否 | - | 套餐：basic / pro / flagship |
| cycle | VARCHAR(16) | 否 | - | 计费周期：month / quarter / year |
| amount | DECIMAL(19,4) | 否 | - | 订单金额（元） |
| status | TINYINT UNSIGNED | 否 | 0 | 订单状态：0-待支付，1-已支付 |
| paid_at | DATETIME(3) | 是 | NULL | 支付完成时间 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_order_order_no (order_no)
- KEY idx_u_order_user_id (user_id)

**说明**：
- `user_id` 关联 `u_user.id`；支付成功后联动写入 `u_user_membership`。
- 当前仅记录"待支付/已支付"两态，无退款/关闭状态，后续如需扩展可在 `status` 上追加。

## u_user_coin_record — 用户创作币流水表

**用途**：记录用户创作币的每一笔收入/支出，是创作币余额（u_user.coin_balance）的明细账。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| biz_no | VARCHAR(64) | 否 | - | 业务唯一编号（幂等键） |
| user_id | BIGINT UNSIGNED | 否 | - | 所属用户 ID，关联 u_user.id |
| biz_type | VARCHAR(32) | 否 | - | 业务类型：leaderboard_reward（榜单奖励）/ admin_adjust（后台调整）/ invite_reward（邀请奖励）等 |
| direction | TINYINT UNSIGNED | 否 | - | 方向：1-收入，2-支出（对应枚举 CoinDirection.INCOME / EXPENSE） |
| amount | DECIMAL(19,4) | 否 | - | 本次金额（始终为正，方向由 direction 表达） |
| balance_after | DECIMAL(19,4) | 否 | - | 本次入账后的余额快照 |
| ref_id | VARCHAR(64) | 是 | NULL | 关联业务 ID（如奖励记录 biz_no、调整工单号） |
| remark | VARCHAR(256) | 是 | NULL | 备注 |
| biz_time | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 业务发生时间 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_user_coin_record_biz_no (biz_no)
- KEY idx_u_user_coin_record_user_time (user_id, biz_time)
- KEY idx_u_user_coin_record_user_type (user_id, biz_type)
- KEY idx_u_user_coin_record_biz_time (biz_time)

**说明**：
- `user_id` 关联 `u_user.id`；`u_user.coin_balance` 为本流水的累计结果，`balance_after` 为写入当时的快照。
- `biz_no` 全局唯一，用于发奖/调整等场景的幂等控制。
- `direction` 取值：1=收入（INCOME），2=支出（EXPENSE）。

## u_earnings_record — 用户收益流水表（通用账本）

**用途**：记录用户在平台内获得的各类收益（风格被使用、里程碑、榜单奖励、邀请奖励等），用于收益列表展示与月度结算。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| user_id | BIGINT UNSIGNED | 否 | - | 收益所属用户 ID，关联 u_user.id |
| type | VARCHAR(32) | 否 | - | 收益类型：USAGE（使用分成）/ MILESTONE（里程碑）/ LEADERBOARD_REWARD（榜单奖励）/ INVITE_REWARD（邀请奖励）/ OTHER |
| source_type | VARCHAR(32) | 是 | NULL | 来源类型：style_market / invite / leaderboard / manual |
| source_id | VARCHAR(64) | 是 | NULL | 上游业务 ID（弱关联，不建外键） |
| title | VARCHAR(128) | 否 | - | 列表展示标题，如「清新」风格被使用 |
| description | VARCHAR(255) | 是 | NULL | 详细描述 |
| amount | DECIMAL(10,2) | 否 | 0.00 | 收益金额（元） |
| status | TINYINT | 否 | 0 | 结算状态：0-未结算，1-已结算 |
| settlement_month | VARCHAR(7) | 否 | - | 归属月份，格式 YYYY-MM，插入时按 created_at 计算 |
| settled_at | DATETIME | 是 | NULL | 实际结算时间 |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | 否 | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引**：
- PRIMARY KEY (id)
- KEY idx_user_status_month (user_id, status, settlement_month)
- KEY idx_user_created (user_id, created_at)
- KEY idx_u_earnings_record_is_deleted (is_deleted)

**说明**：
- `user_id` 关联 `u_user.id`；`source_id` 仅作业务关联，不建外键以便解耦上游。
- `type` 与 `source_type` 均为字符串枚举，由应用层约束取值。
- 月度结算任务按 `settlement_month` + `status=0` 扫描批量打款，打款后回写 `status=1` 与 `settled_at`。

## u_leaderboard_income_submission — 自媒体收入申报记录表

**用途**：用户为参与"自媒体收入榜"提交的月度收入申报，附截图凭证，由管理员审核后计入榜单。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| biz_no | VARCHAR(64) | 否 | - | 业务唯一编号 |
| user_id | BIGINT UNSIGNED | 否 | - | 申报用户 ID，关联 u_user.id |
| period_month | CHAR(7) | 否 | - | 申报所属月份，格式 YYYY-MM |
| amount | DECIMAL(19,4) | 否 | - | 申报金额（元） |
| platform | VARCHAR(64) | 是 | NULL | 自媒体平台：wechat / xiaohongshu / douyin / other |
| screenshot_paths | JSON | 否 | - | 收益截图本地路径列表（JSON 数组，多张） |
| audit_status | TINYINT UNSIGNED | 否 | 0 | 审核状态：0-待审核，1-已通过，2-已拒绝（对应 SubmissionStatus.PENDING / APPROVED / REJECTED） |
| audited_by | BIGINT UNSIGNED | 是 | NULL | 审核管理员 ID |
| audited_at | DATETIME(3) | 是 | NULL | 审核时间 |
| reject_reason | VARCHAR(256) | 是 | NULL | 拒绝原因 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_lis_biz_no (biz_no)
- KEY idx_u_lis_user_status (user_id, audit_status)
- KEY idx_u_lis_status_month (audit_status, period_month)

**说明**：
- `user_id` 关联 `u_user.id`；`audited_by` 关联管理端 `a_admin_user.id`。
- 审核通过后计入 `u_leaderboard_reward_record` 的"自媒体收入榜"候选池。
- `screenshot_paths` 为 JSON 数组，元素为本地磁盘路径（项目无 OSS，存本地）。

## u_leaderboard_reward_record — 榜单奖励发放记录表

**用途**：记录榜单（创作币榜 / 自媒体收入榜）各周期 TOP 用户的奖励发放明细，一次发放一行。

| 字段 | 类型 | 允许空 | 默认值 | 含义 |
| --- | --- | --- | --- | --- |
| id | BIGINT UNSIGNED | 否 | AUTO_INCREMENT | 主键 ID |
| biz_no | VARCHAR(64) | 否 | - | 业务唯一编号 |
| leaderboard_type | TINYINT UNSIGNED | 否 | - | 榜单类型：1-创作币榜，2-自媒体收入榜（月度）（对应 LeaderboardType.COIN / INCOME） |
| period_month | CHAR(7) | 否 | - | 榜单所属月份，格式 YYYY-MM |
| rank_no | INT UNSIGNED | 否 | - | 排名，取值 1-10 |
| user_id | BIGINT UNSIGNED | 否 | - | 获奖用户 ID，关联 u_user.id |
| amount | DECIMAL(19,4) | 否 | 100.0000 | 奖励金额（创作币） |
| coin_record_biz_no | VARCHAR(64) | 是 | NULL | 对应的 u_user_coin_record.biz_no，用于账务对账 |
| granted_by | BIGINT UNSIGNED | 否 | - | 发放管理员 ID |
| granted_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 发放时间 |
| tenant_id | BIGINT UNSIGNED | 否 | 0 | 租户 ID |
| is_deleted | TINYINT UNSIGNED | 否 | 0 | 逻辑删除标记：0-未删除，1-已删除 |
| created_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) | 创建时间 |
| updated_at | DATETIME(3) | 否 | CURRENT_TIMESTAMP(3) ON UPDATE | 更新时间 |
| created_by | BIGINT UNSIGNED | 否 | 0 | 创建人 ID |
| updated_by | BIGINT UNSIGNED | 否 | 0 | 更新人 ID |

**索引**：
- PRIMARY KEY (id)
- UNIQUE KEY uk_u_lrr_biz_no (biz_no)
- UNIQUE KEY uk_u_lrr_type_period_user (leaderboard_type, period_month, user_id) — 同一榜单同一周期同一用户只发一次
- KEY idx_u_lrr_type_period_rank (leaderboard_type, period_month, rank_no)

**说明**：
- `user_id` 关联 `u_user.id`；`granted_by` 关联管理端 `a_admin_user.id`。
- `coin_record_biz_no` 关联 `u_user_coin_record.biz_no`，发放奖励时同步写创作币流水。
- `leaderboard_type` 取值：1=创作币榜（COIN），2=自媒体收入榜（INCOME，月度）。
