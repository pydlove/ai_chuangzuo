# 创作页字数设置弹窗方案

**日期**: 2026-06-26  
**状态**: 待实现  
**关联原型**: full-prototype-v20.html

---

## 1. 功能概述

将创作页当前的「字数要求」`<select>` 下拉框（固定 800 / 1500 / 2500 三档）替换为弹窗选择器，支持 4 个维度的字数设置：按平台推荐、按内容场景、按字数档位、自定义字数。每个选项附带适用场景说明，确认后回写到创作页按钮。

## 2. 弹窗结构

### 2.1 整体布局

- **标题**：设置文章字数
- **副标题**：选择合适的字数，让 AI 写出更精准的内容
- **顶部标签栏**：4 个标签切换
- **内容区**：根据选中标签渲染对应的选项列表
- **底部按钮**：取消 / 确认

### 2.2 标签栏（4 个维度）

| 标签 | key | 默认选中场景 |
|---|---|---|
| 按平台推荐 | `platform` | 当创作页已选「发布平台」时默认跳此标签 |
| 按内容场景 | `scenario` | — |
| 按字数档位 | `tier` | 未选平台时的默认标签 |
| 自定义字数 | `custom` | — |

## 4. 维度 2：按内容场景

4 个常见内容场景：

| 场景 | 字数 | 适用说明 |
|---|---|---|
| 教程 / 步骤 | 1200 字 | 操作步骤详细说明，适合图文对照 |
| 测评 / 对比 | 1000 字 | 优缺点详细对比，附评分 |
| 清单 / 种草 | 500 字 | 快速清单 + 标签，重点突出 |
| 故事 / 叙事 | 1800 字 | 沉浸式叙事，节奏完整 |

## 5. 维度 3：按字数档位

5 个字数档位：

| 档位 | 字数 | 适用说明 |
|---|---|---|
| 短文 | 500 字 | 速读，3 分钟读完 |
| 中等 | 1000 字 | 标准阅读，5 分钟 |
| 标准 | 1500 字 | 深度阅读，8 分钟 |
| 长文 | 2500 字 | 专题阅读，12 分钟 |
| 超长 | 3000 字 | 深度专题（上限） |

## 6. 维度 4：自定义字数

- **数字输入框**：1-3000 字，输入超出范围自动夹紧
- **滑块**：1-3000 范围，可拖动
- **实时预览文字**：AI 将生成约 X 字的文章
- **应用按钮**：仅在输入有效数字（1-3000）时激活
- 默认显示当前已选字数

## 7. 选项卡片样式

每个选项（除自定义外）以卡片形式展示：

- 左上：字数 + 单位（如 "1500 字"）
- 右上：选中态用绿色勾选图标
- 下方：适用场景说明（如 "标准深度文"）
- 卡片背景：未选中 #fff，选中 #f6ffed，边框 #07c160

## 8. 创作页集成

### 8.1 旧 UI（删除）

```html
<select class="mock-input">
  <option>800 字左右</option>
  <option>1500 字左右</option>
  <option>2500 字左右</option>
</select>
```

### 8.2 新 UI

按钮组件：

```html
<button class="word-count-trigger" onclick="openWordCountModal()">
  📝 <span id="current-word-count-label">1500 字左右 · 标准深度文</span> ✏️
</button>
```

按钮样式与现有「+ 风格库」按钮风格保持一致（白底、绿字、可点击态高亮）。

### 8.3 联动逻辑

- 弹窗中确认后，按钮文字立即更新
- 已选项用全局状态 `currentWordCount: number` + `currentWordLabel: string` 维护
- 联动到 mock-form 提交时的字数要求

## 9. 默认值与首次打开

- `currentWordCount` 默认 1500
- `currentWordLabel` 默认 "标准深度文"
- 首次打开弹窗时按 §2.2 的默认规则选中标签
- 默认选项与当前 `currentWordCount` 匹配的项设为选中态

## 10. 弹窗交互细节

- **关闭按钮**：右上角 X（点击关闭不应用）
- **底部按钮**：取消（关闭不应用）/ 确认（应用并关闭）
- **移动端**：弹窗全屏，标签栏支持横滑
- **选项点击**：立即更新选中态（无需等确认）

## 12. 技术要点

- 纯前端 HTML/CSS/JS 原型，无后端依赖
- 弹窗 DOM 通过 JavaScript 动态生成，与现有模板库弹窗风格保持一致
- 选项点击后只更新选中态，确认按钮点击后才回写到 `currentWordCount`
- 自定义字数输入框需做边界检查（1-3000）
- 移动端复用同一弹窗组件，自适应全屏布局
- 与发布平台选择器联动：监听发布平台变更，决定下次打开弹窗时的默认标签
