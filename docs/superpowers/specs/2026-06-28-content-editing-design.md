# 生成内容手动编辑设计文档

## 背景

当前 `preview.html` 中的文章内容是写死的静态 HTML。用户在预览 AI 生成结果时，发现个别字词、段落或标题不合适，希望像编辑在线文档一样直接修改，而不是重新生成整篇文章。

## 目标

- 让普通用户无需理解 HTML，即可修改预览页中的文字内容。
- 提供轻量内联编辑和独立编辑页两种入口。
- 用户手动保存后，修改内容持久化在本地，刷新页面不丢失。
- 本期不引入 AI 二次改写、不增删段落、不做版本历史。

## 范围

**本期做：**

- 预览页 `preview.html` 开启/关闭编辑态。
- 可编辑元素：文章标题、小标题、正文段落、重点高亮块、列表项。
- 悬浮保存/取消操作栏。
- hover 单元素编辑图标 + 顶部批量编辑开关。
- 独立编辑页 `edit.html`。
- 编辑结果存 `localStorage`。
- 粘贴时过滤样式，仅保留纯文本。

**本期不做：**

- 增删段落、调整段落顺序。
- 修改发布描述、推荐标签、模板样式。
- AI 辅助改写。
- 撤销/恢复原始版本。
- 多设备同步。

## 用户流程

1. 用户在 `preview.html` 看到生成结果，发现某段文字不合适。
2. 点击顶部「编辑内容」开关，或 hover 到具体元素点击「编辑」图标，该元素进入可编辑状态。
3. 直接输入修改。
4. 改完后点击底部悬浮栏「保存修改」。
5. 页面退出编辑态，显示最新内容，并轻提示「内容已保存」。
6. 若需要批量修改，点击「进入编辑模式」跳转到 `edit.html`，改完保存后返回。

## 数据模型

编辑结果以 JSON 对象存储：

```json
{
  "articleId": "default_preview",
  "savedAt": 1712345678901,
  "blocks": [
    { "type": "title", "html": "如何高效管理时间：从混乱到掌控的 5 个方法" },
    { "type": "paragraph", "html": "时间对每个人来说都是公平的..." },
    { "type": "heading", "html": "01｜优先级排序：先做重要的事" },
    { "type": "paragraph", "html": "很多人一早打开手机就被消息牵着走..." },
    { "type": "highlight", "html": "<strong>关键结论：</strong> 管理时间本质是管理注意力..." }
  ]
}
```

字段说明：

- `articleId`：当前固定为 `"default_preview"`，后续接入真实文章 ID 后可替换。
- `savedAt`：保存时间戳。
- `blocks`：按 DOM 顺序存储每块编辑后的 `innerHTML`。
  - `type`：`title` / `heading` / `paragraph` / `highlight` / `list-item`。
  - `html`：该块的最终 HTML 内容。

存储：

- `localStorage` 键名：`aichuangzuo_article_edits`
- 读写时捕获异常，避免 localStorage 不可用时阻塞页面。

## 渲染方案

1. **预览页加载时**：
   - 先按现有逻辑渲染静态 HTML。
   - 调用 `loadArticleEdits()` 读取 `localStorage`。
   - 若存在已保存 edits，按 `blocks` 顺序替换对应可编辑元素的内容。

2. **进入编辑态**：
   - 调用 `enableArticleEditing(mockupEl)`。
   - 给 `.preview-title`、`.preview-heading`、`.article-preview > p`、`.preview-highlight`、`.preview-list li` 添加 `contenteditable="true"` 和编辑样式 class。
   - 在内存中保存「编辑前快照」，用于取消时恢复。

3. **保存时**：
   - 遍历可编辑元素，提取当前 `innerHTML`。
   - 过滤空内容，组装成 `blocks`。
   - 写入 `localStorage`。
   - 关闭 `contenteditable`，移除编辑样式。

4. **独立编辑页 `edit.html`**：
   - 加载时读取 `loadArticleEdits()`。
   - 用 `renderEditBlocks()` 把 blocks 渲染成一排可编辑区域。
   - 保存后写入 `localStorage` 并返回 `preview.html`。

## UI 设计

### 预览页

- PC 和移动端预览区顶部工具栏新增：
  - 「编辑内容」开关按钮。
  - 「进入编辑模式」按钮（跳转 `edit.html`）。
- 开启编辑态后：
  - 可编辑元素显示淡绿色边框。
  - hover 时显示小「编辑」图标。
  - 已修改但未保存的元素显示橙色小点。
- 底部悬浮操作栏：
  - 左侧显示「已修改 N 处」。
  - 右侧「取消」「保存修改」。

### 独立编辑页 `edit.html`

- 顶部：返回按钮 + 页面标题「编辑内容」。
- 中部：按文章结构展示所有可编辑块，每块一个 `contenteditable` 区域。
- 底部：「取消」「保存修改」按钮。

## 错误处理

- **空内容**：保存时若标题为空，阻止保存并提示「标题不能为空」。
- **粘贴脏内容**：监听 `paste` 事件，强制纯文本粘贴，过滤外部样式。
- **localStorage 失败**：捕获异常，轻提示「保存失败，请检查浏览器存储权限」。
- **DOM 结构变化**：若静态 HTML 后续升级导致 block 数量不一致，按索引匹配；匹配不上的块保留原始内容。

## 测试与验证

- 手动验证：进入编辑态修改标题、段落、高亮块，保存后刷新页面，确认内容保留。
- 手动验证：点击「取消」后修改不保留。
- 手动验证：从 `preview.html` 进入 `edit.html` 修改并保存，返回后预览更新。
- 边界验证：标题置空保存时被阻止。
- 边界验证：粘贴带样式的文本时仅保留纯文本。
- 边界验证：localStorage 禁用时提示用户。

## 后续可扩展

- AI 辅助改写：选中文字后弹出「改得更口语化 / 更正式 / 更简短」等选项。
- 恢复原始版本：保存一份 `originalBlocks` 用于一键恢复。
- 增删与排序段落：为每个块增加上移/下移/删除/新增按钮。
- 图片与封面编辑：允许替换文章中的示例图片。
