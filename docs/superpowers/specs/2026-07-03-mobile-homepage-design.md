# 首页移动端适配设计

**日期**: 2026-07-03
**状态**: 已确认，待实现
**关联文件**: `project/user/web/src/views/Home.vue`

---

## 1. 功能概述

针对用户端首页（`/`）做手机端访问适配。当前首页为桌面端布局，在窄屏下顶部导航挤压、Hero 标题过大、功能卡片四列过窄、步骤横向排列导致可读性差。本次改造在保持现有桌面样式和黑暗主题的前提下，通过 CSS 媒体查询为手机端（≤768px）提供独立的布局与交互。

---

## 2. 设计决策

- **改造范围**：仅修改 `project/user/web/src/views/Home.vue`，不新增路由、不改动其他页面。
- **断点选择**：`@media (max-width: 768px)`，与项目内现有移动端适配（玩法指南、收益排行榜等）保持一致。
- **头部方案**：手机端将导航链接折叠到右侧抽屉菜单，桌面端保持原样。
- **CTA 处理**：手机端头部仍外露「开始创作」按钮，避免核心转化入口被隐藏。
- **内容策略**：不删减内容，仅调整排列方式、字号、间距，保持信息完整。
- **主题兼容**：所有新增/覆盖样式同步支持 `body[data-theme="dark"]`。

---

## 3. 页面结构

首页仍由以下区块组成，各区块在手机端按同一顺序垂直排列：

1. 顶部导航（含抽屉）
2. Hero 区
3. 数据区
4. 特色功能区
5. 使用步骤区
6. 最终 CTA 区
7. 底部

### 3.1 顶部导航

**桌面端（>768px）**

```
[Logo  爱创作]        [首页] [会员] [玩法指南] [主题] [开始创作]
```

**手机端（≤768px）**

```
[Logo  爱创作]                              [开始创作] [≡]
```

点击汉堡按钮 `≡` 从右侧滑出抽屉：

```
┌─────────────────────┐
│ 首页                │
│ 会员                │
│ 玩法指南            │
│ 🌙/☀ 切换主题       │
└─────────────────────┘
```

抽屉行为：

- 遮罩层覆盖页面，点击遮罩或点击抽屉内链接后关闭
- 抽屉宽度 260px，从右侧滑入
- 打开时禁止底层页面滚动
- 过渡动画 0.25s ease

## 5. 交互说明

### 5.1 抽屉状态

- 使用 `ref` 定义 `mobileMenuOpen`，控制抽屉显示。
- 点击汉堡按钮切换状态。
- 点击遮罩层或抽屉内任意链接后，设置 `mobileMenuOpen = false`。
- 监听 `mobileMenuOpen`：打开时给 `document.body` 添加 `overflow: hidden`，关闭时移除。

### 5.2 主题切换

- 抽屉内的主题切换按钮功能与桌面端一致，调用现有 `toggleTheme`。
- 图标随 `currentTheme` 变化。

### 5.3 路由跳转

- 抽屉内的链接使用 `router-link`，点击后自动关闭抽屉。

## 6. 视觉规范

沿用现有设计系统：

- 主色：`#FF2442`
- 背景：`#fff`（浅色）、`#141414`（深色）
- 文字：`#1a1a1a` / `#595959`（浅色），`#e0e0e0` / `#a6a6a6`（深色）
- 抽屉背景：`#fff`（浅色）、`#1f1f1f`（深色）
- 遮罩：`rgba(0, 0, 0, 0.3)`
- 圆角：卡片 `12px`，按钮 `24px`，抽屉 `0`（贴边）
- 字号：参考「响应式规则」章节

## 7. 验收标准

- [ ] 在 375px 宽度下，顶部导航不挤压、不溢出。
- [ ] 汉堡按钮可点击，抽屉正常滑出/关闭。
- [ ] 抽屉内链接可正常跳转并关闭抽屉。
- [ ] Hero、数据、功能、步骤、CTA、底部在手机端布局合理，无重叠或截断。
- [ ] 桌面端样式与改造前保持一致。
- [ ] 黑暗主题下手机端各元素颜色正确。
- [ ] 抽屉打开时底层页面不可滚动。

## 8. 不做的范围

- 不新增路由或页面。

- 不抽成公共导航组件（本次聚焦首页验证效果）。
- 不引入第三方响应式库或 UI 框架新组件。
- 不添加底部固定 Tab 栏或 PWA 相关能力。
