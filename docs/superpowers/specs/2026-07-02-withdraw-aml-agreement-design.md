# 提现页面反洗钱/合规协议勾选设计

## 背景

「创作币 & 提现」页面需要增加一个协议勾选环节，用于明确用户提现行为的合规义务，防范洗钱、套现、虚假交易等违法行为。

## 目标

- 在用户提交提现申请前，强制其阅读并同意《提现服务协议》。
- 通过本地存储记住用户的同意状态，减少重复操作。
- 提供清晰的协议全文查看入口。

## 设计方案

### 交互位置

在「申请提现」弹框（`coin-apply-panel`）底部、操作按钮区域上方增加协议勾选行。

```
[ ] 我已阅读并同意《提现服务协议》
```

- 勾选框未选中时，「提交申请」按钮禁用。
- 点击《提现服务协议》打开协议全文弹框。
- 若用户未勾选即点击提交按钮，提示「请先阅读并同意《提现服务协议》」。

### 协议全文弹框

- 标题：提现服务协议
- 宽度：560px，居中显示
- 内容：使用有序列表或段落展示协议条款
- 底部：「* 本协议内容仅供参考，具体以平台最终公示为准。」

### 协议条款（通用模板，需法务审核）

1. 用户承诺提现资金来源合法，不得利用本平台进行洗钱、套现、赌博、诈骗等违法活动。
2. 用户申请提现的账户信息（支付宝账号、真实姓名）必须与本人实名认证信息一致。
3. 用户应确保邀请好友行为真实有效，禁止通过虚假注册、刷单、机器刷量等方式获取创作币。
4. 平台有权对异常提现行为进行审核、延迟到账、拒绝提现或冻结相关收益。
5. 因用户提供错误账户信息、账户异常或违反法律法规导致的提现失败，平台不承担任何责任。
6. 平台可根据法律法规及业务需要调整本协议内容，调整后会通过官方渠道通知用户。

### 状态持久化

- 使用 `localStorage` 键 `aichuangzuo_withdraw_agreement_accepted` 保存用户同意状态。
- 值为 `true` 时，下次打开「申请提现」弹框默认勾选。
- 用户仍可手动取消勾选，但取消后无法提交申请。

### 校验逻辑

- `canSubmitApply` 增加条件：`agreementAccepted.value === true`。
- 提交函数 `submitApply` 中先校验勾选状态，未勾选时调用 `message.warning` 提示。

## 影响范围

  - 模板：申请弹框内增加勾选行和协议弹框

  - 样式：增加勾选行和协议弹框样式
