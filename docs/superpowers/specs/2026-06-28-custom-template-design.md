# 自定义导出模板设计文档

## 背景

当前创作页和预览页共用一份硬编码的 `templatePresets` 导出模板列表。随着平台覆盖公众号、小红书、今日头条、百家号、知乎、抖音图文，模板数量已经达到 30 个，但仍无法满足用户对品牌色、个人排版风格的差异化需求。

本设计在保持「纯前端原型、无后端、localStorage 存储」的约束下，增加低门槛的自定义模板能力。

## 目标

- 让普通用户无需理解 HTML/CSS，也能创建自己的导出模板。
- 自定义模板与普通预设模板在预览、导出流程中表现一致。
- 自定义模板数据持久化在本地，刷新页面不丢失。

## 范围

**本期做：**

- 在「导出模板库」弹窗中新增「我的模板」分类。
- 提供「创建自定义模板」入口。
- 基于现有预设模板，通过 4 组可视化选项快速定制外观。
- 保存、应用、重命名、删除自定义模板。
- 自定义模板存 `localStorage`。

**本期不做：**

- 直接编辑 HTML/CSS。
- 模板分享、导入、导出。
- 云端同步或多端同步。
- 字体选择、复杂动画、响应式断点控制。

## 用户流程

1. 用户在创作页点击「+ 模板库」。
2. 弹窗顶部出现「全部 / 公众号 / 小红书 / ... / 我的模板」分类。
3. 切换到「我的模板」：
   - 若为空，显示占位提示和「创建自定义模板」按钮。
   - 若已有模板，列出用户保存的模板；点击卡片即应用该模板，hover 时卡片右上角显示「编辑 / 删除」图标。
4. 点击「创建自定义模板」进入新建面板：
   - 左侧大预览实时显示效果。
   - 右侧选择「基于哪个预设」。
   - 依次选择配色主题、标题样式、重点高亮、是否启用卡片段落。
   - 输入模板名称。
   - 点击「保存到我的模板」。
5. 保存后自动返回「我的模板」列表，新模板置顶。
6. 用户点击任意模板（自定义或预设）即可应用到预览页。

## 数据模型

自定义模板以 JSON 对象存储，结构如下：

```json
{
  "id": "custom_1712345678901",
  "name": "我的小红书种草风",
  "baseKey": "xiaohongshu",
  "overrides": {
    "theme": "pink",
    "titleStyle": "center",
    "highlightStyle": "border",
    "useCards": false
  },
  "createdAt": 1712345678901
}
```

字段说明：

- `id`：本地唯一标识，前缀 `custom_` 加时间戳。
- `name`：用户输入的模板名称，限制 20 字。
- `baseKey`：基于哪个预设模板，对应 `templatePresets` 的 `key`。
- `overrides`：4 组覆盖配置。
  - `theme`：`brand` / `blue` / `red` / `gray` / `pink` / `orange`
  - `titleStyle`：`center` / `left` / `underline`
  - `highlightStyle`：`border` / `background` / `quote`
  - `useCards`：`true` / `false`
- `createdAt`：创建时间戳，用于排序。

存储：

- `localStorage` 键名：`aichuangzuo_custom_templates`
- 值：自定义模板数组的 JSON 字符串。
- 读写时捕获异常，避免 localStorage 不可用时阻塞页面。

## 渲染方案

自定义模板不新增独立的渲染函数，而是复用现有 `applyTemplateToPreview(mockupEl, templateKey)`，并在其后追加一层「样式覆盖」。

实现步骤：

1. 加载页面时，从 `localStorage` 读取自定义模板数组，与 `templatePresets` 合并为运行时模板列表 `runtimeTemplates`。
2. 合并规则：
   - 自定义模板排在「我的模板」分类最前。
   - 自定义模板与预设模板通过 `key` 区分，自定义模板 `key` 使用其 `id`。
3. `applyTemplateToPreview` 先按 `baseKey` 应用预设模板的基础样式。
4. 若目标模板是自定义模板，再调用 `applyTemplateOverrides(mockupEl, overrides)`，根据 `overrides` 修改变量。需要在预设模板渲染逻辑中先引入 CSS 变量（如 `--template-primary`、`--template-bg`、`--template-border`），`applyTemplateOverrides` 通过修改这些变量或追加 class 实现覆盖：
   - 配色：覆盖 CSS 变量值。
   - 标题样式：给文章标题追加对应 class。
   - 重点高亮：给 `.preview-highlight` 元素追加对应 class。
   - 卡片段落：给正文段落追加 `.card-paragraph`。
5. 预览页和导出渲染都走同一套逻辑，保证所见即所得。

## UI 设计

### 模板库弹窗

- 分类 tab 增加「我的模板」，置于最右侧。
- 「我的模板」列表顶部常驻「+ 创建自定义模板」按钮。
- 自定义模板卡片与预设模板卡片样式一致，但在左上角增加一个「我」小标签。
- 每个自定义模板卡片 hover 时显示「编辑 / 删除」图标按钮。点击「编辑」重新进入创建面板并回填当前配置。

### 创建/编辑面板

面板为弹窗内的子页面，宽度与模板库弹窗一致：

- 左侧 55%：大预览区，基于当前选择实时渲染。
- 右侧 45%：配置区。
  - 第一步：「基于模板」——从预设模板中选择基础。
  - 第二步：「配色主题」——6 个圆形色块，选中带对勾。
  - 第三步：「标题样式」——3 个缩略图卡片。
  - 第四步：「重点高亮」——3 个缩略图卡片。
  - 第五步：「段落卡片」——单个开关。
  - 第六步：「模板名称」——输入框，必填，最多 20 字。
  - 底部：「取消 / 保存」。

所有选项切换时，左侧预览即时更新，无需额外确认。

## 错误处理

- 名称不能为空或纯空格，保存时校验并提示。
- 名称超过 20 字时截断或禁止输入。
- `localStorage` 写入失败时显示轻提示「保存失败，请检查浏览器存储权限」。
- 读取到损坏的自定义模板数据时，忽略损坏项，保留可解析项。
- 删除前二次确认：「确定删除「xxx」？删除后不可恢复」。

## 测试与验证

- 手动验证：创建 1 个自定义模板，切换配色/标题/高亮/卡片，确认左侧预览实时变化。
- 手动验证：保存后进入预览页，确认文章按自定义模板渲染。
- 手动验证：刷新页面后，「我的模板」列表仍保留。
- 手动验证：删除自定义模板后，列表和 localStorage 同步清空。
- 边界验证：localStorage 满或禁用时，页面不报错，提示用户。

## 后续可扩展

- 模板导入/导出（JSON 文件）。
- 更多版式组件（封面图、作者简介、分隔线样式）。
- 自定义模板与发布平台绑定，选择平台时自动推荐对应自定义模板。
