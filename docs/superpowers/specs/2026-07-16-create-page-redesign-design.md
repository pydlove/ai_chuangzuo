# 创作页双模式重设计

> 用户端「开始创作」页重做：默认**对话引导模式**（固定问题流，新手/没灵感友好），可切换**极简模式**（居中一体化输入卡片，熟手高效）。两模式共享同一份创作配置，localStorage 记住模式选择。队列从常驻右侧栏改为右侧抽屉。移动端双模式均适配。

## 背景与目标

现状问题（PC 1440px 实测截图诊断）：

- 标题框 + 正文框两个分离灰框，表单感重，无 AI 产品感
- 正文 textarea 默认占近半屏，空状态大片空白
- 平台/字数/风格/模板 4 个配置胶囊权重低、无图标，无数据时缩成小点
- 右侧队列栏空状态浪费一大块空间
- 纯白卡片 + 纯灰背景，品牌红只出现在按钮上，气质偏后台工具

目标：

- 新手进页面不被空白表单吓到，AI 一步步引导完成创作配置
- 熟手一键切到高效输入面板，聚焦输入
- 双模式共享配置状态，切换无损失
- 视觉有品牌氛围（淡红光晕、聚焦微光），脱离后台感

## 已确认的关键决策

| 决策点 | 结论 |
|---|---|
| 对话实现方式 | 前端固定问题流（3 问 + 确认），不需要后端 AI 多轮能力 |
| 默认模式 | 默认引导模式；切换后写 localStorage，之后进页面记住选择 |
| 快捷胶囊交互 | 两段式：点选 → 内联效果预览卡 → 点确认才提交 |
| 极简模式队列 | 右侧抽屉（a-drawer），顶部按钮触发，点生成后自动滑出 |
| 移动端 | 双模式都做移动端适配 |
| 对话模式草稿 | 不做保存草稿（轻量流程），草稿能力仅极简模式 |

## 组件拆分

现 `CreateIndex.vue`（3857 行）拆分为：

```
project/user/web/src/views/console/create/
├── CreateIndex.vue        # 薄容器：模式切换 + 共享状态 + 额度
├── GuidedChat.vue         # 对话引导模式
├── ChatMessage.vue        # 单条消息（AI 气泡 / 用户气泡 / 进度卡片 / 结果卡片）
├── QuickReplies.vue       # 快捷胶囊组（两段式：点选→效果卡→确认）
├── MinimalPanel.vue       # 极简模式输入面板
├── QueueDrawer.vue        # 队列抽屉（两模式共用）
└── TopicCapsules.vue      # 灵感胶囊（引导模式步骤1快捷回复 + 极简模式卡片下方）
```

现有 4 个配置弹框（平台/字数/风格/模板）原样保留为独立组件复用：极简模式配置胶囊弹它们；对话模式确认步骤「改配置」也弹同一套。

## 模式切换机制

- 状态：`createMode: 'guided' | 'minimal'`
- 持久化：localStorage `aichuangzuo_create_mode`；读取失败/无值 fallback `guided`
- 入口：
  - 引导模式顶部右上角：`熟手模式 →`
  - 极简模式顶部：`💬 引导模式`
- 切换只切组件，**配置不丢**：平台、字数、风格、模板、主题、要求是两模式共享的同一份状态（Pinia store 或 CreateIndex provide/inject）

## 对话引导模式

### 页面形态

整页居中聊天流（max-width ~720px），顶部一行：`开始创作` + 右侧 `熟手模式 →`。

### 固定问题流（4 步）

**步骤 1 · 主题**
- AI 气泡：「想写一篇什么主题的文章？」+ 输入框（回车或点发送）
- 输入框下方灵感胶囊快捷回复（复用 TopicCapsules 数据）：「没灵感？点一个」+ 换一批
- 点灵感胶囊 = 标题填入主题 + 概要并入要求（与现有极简模式 applyTopic 逻辑一致）

**步骤 2 · 平台**
- AI 气泡：「准备发哪个平台？」+ 平台胶囊（公众号/小红书/头条/知乎/百家号/抖音/通用）
- 两段式交互：点胶囊 = 选中 + 下方展开效果预览卡（不提交）；点预览卡「确认 ✓」才提交进入下一步；再点别的胶囊预览卡内容跟着换
- 平台预览卡内容：推荐字数、默认模板、平台特性（标签/emoji/段落风格，如「小红书：推荐 800 字，短段落多 emoji，自动带 5 个话题标签」）
- 确认后自动带出该平台推荐字数和默认模板

**步骤 3 · 风格**
- AI 气泡：「想要什么风格？」+ 风格胶囊（系统预设前 6 个 + 「我的风格 ›」入口）
- 同样两段式：风格预览卡展示风格描述 + 提示词摘要（现有 `promptSummary`）

**步骤 4 · 确认**
- AI 汇总卡片：标题（主题）、平台、字数、风格、模板、「本次消耗 1 次 · 剩余 N 次」
- 操作：`[⚡ 开始生成]` `[改主题]`（回步骤 1，已答内容保留）`[改配置]`（弹现有配置弹框）

### 生成中与完成

- 点「开始生成」后，确认卡片原地变进度卡片：进度条 + 阶段文案（正在生成大纲…正文…排版）
- 完成后变结果卡片：`✅ 已生成完成` + `[查看文章]`（跳作品/预览）`[再写一篇]`（清空对话回步骤 1，配置保留）

### 异常处理

- 额度为 0：步骤 1 之前 AI 气泡提示「本月额度已用完」+ `[去充值]`，不出输入框
- 生成失败：进度卡片变失败态 + `[重试]`（沿用上次配置重新提交）
- 任意步骤可点顶部「熟手模式」跳走，已选配置带过去

## 极简模式

### 布局

无右侧栏，整页居中（max-width ~760px）：

```
┌────────────────────────────────────────────┐
│ 开始创作        本月剩余 12 次 [📋队列②] [💬引导模式] │
│ ┌─ 一体化输入卡片 ────────────────────────┐ │
│ │ 输入标题或想法…            （18px 无边框） │ │
│ │ 补充要求：语气、案例、重点…（无边框，      │ │
│ │ 默认 4 行高，输入自动增高，最长 300px）    │ │
│ │ ───────────────────────────────────── │ │
│ │ [📱公众号▾][📝1500字▾][🎨风格▾][🖼模板▾] │ │
│ │ 保存草稿 · 草稿箱            [⚡ 生成文章] │ │
│ └────────────────────────────────────────┘ │
│ 没灵感？ [35岁被裁…][租房改造…][AI工具…] 换一批 │
└────────────────────────────────────────────┘
```

相比现状的变化：

- 标题框 + 正文框合并成一张卡片，控件全部内嵌
- textarea 默认收紧（~120px），输入时自动撑高（上限 300px）
- 配置胶囊加图标、加大一号，4 个平铺
- 生成按钮内嵌卡片右下角
- 保存草稿/草稿箱保留为卡片底部左侧小链接

### 视觉升级（两模式通用）

- 页面背景：顶部品牌淡红 radial-gradient 光晕（#ff2442 约 5% 透明度）
- 输入卡片聚焦态：红色微光描边/阴影
- AI 气泡左侧小 avatar（logo 图标）

### 队列抽屉

- 顶部 `[📋 队列②]` 按钮，徽章数 = 进行中任务数
- 点击右侧滑出 a-drawer（宽 360px），内容复用现有 queue-panel 列表样式与逻辑
- **点「生成文章」后抽屉自动滑出**（异步任务需立刻看到进度反馈）；任务完成后不自动关，用户手动关
- 移动端抽屉全屏

## 移动端适配

### 引导模式

- 消息列全宽（padding 16px），AI 气泡 max-width 85%
- 快捷胶囊横向滑动（不换行挤压）
- 效果预览卡全宽
- 输入框固定底部（类微信聊天），避免键盘顶起时布局跳动

### 极简模式

- 输入卡片全宽，标题/正文区不变
- 配置胶囊横向滑动
- 顶部行压缩：标题 + `[📋队列②]` + `[💬]`，额度数字并入队列抽屉顶部
- 队列抽屉全屏滑出

## 测试方案

沿用现有 Playwright E2E 模式（`tests/e2e/`）：

1. `verify_create_mode_switch.py` — 默认进引导模式；点「熟手模式」切到极简；刷新后记住选择
2. `verify_guided_flow.py` — 引导全流程：输主题 → 点平台胶囊出效果卡 → 确认 → 选风格 → 确认卡片生成 → 进度卡片出现
3. `verify_minimal_generate.py` — 极简模式点生成 → 抽屉自动滑出 → 队列出现任务
4. 配置共享验证：引导里选小红书，切极简后胶囊显示小红书

## 不做（YAGNI）

- 真 AI 多轮对话（后端无意图提取能力，固定问题流已满足引导诉求）
- 对话模式保存草稿
- 向导分步式（步骤条形态被对话形态取代）
- 选题卡片墙为主页面（灵感胶囊在两模式中保留，但不翻转主次）
