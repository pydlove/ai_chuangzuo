# 消息通知中心设计

## 背景

爱创作原型目前仅在 `settings.html` 中提供「消息通知」开关（生成完成通知、新功能上线通知、营销优惠活动通知），但没有统一的消息中心，用户无法查看历史消息。本设计新增一个轻量级站内消息中心，支持四类消息的历史记录查看。

## 目标

- 让用户能集中查看四类消息的历史记录。
- 保持与现有原型一致的纯前端实现，使用 `localStorage` 持久化。
- 不引入后端或额外依赖。

## 消息类型

| 分类 | 类型标识 | 触发时机 |
|---|---|---|
| 生成完成 | `generation` | 生成队列中任务状态变为 `completed` 时自动写入 |
| 会员提醒 | `membership` | 根据模拟会员到期日，在到期前 7 天、3 天、当天及续费成功时写入 |
| 新功能 | `feature` | `notifications.html` 首次加载时，根据版本号写入预设消息 |
| 优惠活动 | `promotion` | `notifications.html` 首次加载时，根据活动期写入预设消息 |

## 入口与未读提示

- 在 `.prototype-nav` 右上角新增铃铛图标。
- 有未读消息时，铃铛右上角显示红色数字角标；超过 99 显示「99+」。
- 点击进入 `notifications.html`。
- 进入消息中心后所有消息自动标记为已读，返回其他页面时角标消失。
- 移动端因导航栏空间有限，铃铛入口暂仅用于 PC 端；移动端可通过个人中心进入。

## 消息中心页结构

页面路径：`.superpowers/brainstorm/6491-1782131242/content/notifications.html`

- 顶部四个 Tab：「生成完成」「会员提醒」「新功能」「优惠活动」。
- 每个 Tab 内消息按时间倒序排列。
- 单条消息展示：标题、摘要、时间；未读消息标题加粗或左侧加绿色竖条。
- Tab 栏右侧提供「清空本类消息」按钮，点击后二次确认，确认后删除该分类全部消息。
- 空状态：某类无消息时显示「暂无消息」。

## 数据模型

`localStorage` key：`aichuangzuo_notifications`

```json
[
  {
    "id": "uuid",
    "type": "generation|membership|feature|promotion",
    "title": "文章生成完成",
    "summary": "《小红书爆款标题技巧》已生成，点击预览",
    "read": false,
    "createdAt": "2026-06-29T10:30:00"
  }
]
```

## 交互规则

- 自动已读：切换到某个 Tab 时，该 Tab 下所有未读消息自动标记为 `read=true`。
- 一键清空：删除当前分类的全部消息，操作前需二次确认。
- 点击消息：
  - `generation`：跳转 `preview.html` 或 `works.html`。
  - `membership`：跳转 `pricing.html`。
  - `feature` / `promotion`：根据内容跳转对应页面，暂无则仅标记已读。
- 角标实时更新：`shared.js` 提供 `updateNotificationBadge()`，在页面加载和产生新消息时调用。

## 与现有功能的关系

- 复用 `shared.js` 中的 `showToast` 作为新消息到达时的即时提醒。
- 复用 `settings.html` 中已有的通知开关设置；若用户关闭某类通知，则不再写入对应类型消息。
- 生成完成通知与现有 `submitGenerationTask` / 生成队列流程打通。

## 非目标

- 不实现消息推送（Web Push / 短信 / 邮件）。
- 不实现单条删除、手动标记已读 / 未读。
- 不实现消息搜索、筛选（除四类 Tab 外）。
- 不实现后端消息持久化或跨设备同步。

## 后续可扩展
