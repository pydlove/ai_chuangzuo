# 邀请有礼活动设计

## 背景

爱创作用户端采用邮箱注册 + 会员订阅模式。为了以较低成本获取新用户，同时激励老用户主动传播，设计「邀请有礼」长期功能。该功能入口常驻个人中心，通过「会员天数 + 创作币 + 提现」三层激励，把拉新、付费转化、用户留存串联起来。

## 目标

1. **拉新注册**：老用户邀请新用户完成邮箱注册，双方获得奖励。
2. **促进付费**：被邀请人购买会员后，邀请人获得创作币返利。
3. **长期激励**：创作币可抵扣会员购买，满 100 创作币可提现到支付宝，形成持续传播动力。

## 奖励规则

### 邀请人奖励

| 累计有效邀请人数 | 奖励 |
|---|---|
| 3 人 | 3 天专业版会员 |
| 5 人 | 5 天专业版会员 |
| >5 人后，每多 1 人 | 额外 2 天专业版会员 |

示例：

- 邀请 3 人 → 3 天专业版会员
- 邀请 5 人 → 5 天专业版会员（累计 8 天专业版会员）
- 邀请 6 人 → 再 +2 天专业版会员（累计 10 天专业版会员）

### 被邀请人奖励

- 通过邀请链接或邀请码注册，并完成邮箱验证后，获得 **5 个创作币**。
- 该奖励仅面向被邀请的新用户，自然注册用户不享受。

### 购买返利

| 被邀请人行为 | 邀请人奖励 |
|---|---|
| 首次购买会员 | 订单实付金额 × 10% 创作币，向上取整 |
| 后续续费或再次购买会员 | 订单实付金额 × 5% 创作币，向上取整 |

创作币与人民币 1:1 锚定，可在购买会员时抵扣现金，也可满 100 后提现。

## 有效邀请定义

一个邀请被认定为「有效邀请」需同时满足：

1. 被邀请人通过邀请链接或邀请码完成注册。
2. 注册邮箱完成验证。
3. 邮箱域名不在临时邮箱黑名单内。
4. 通过设备 / IP / 行为风控校验。

## 入口与邀请流程

### 用户端入口

1. **个人中心 / 设置页**：常驻「邀请有礼」菜单项。
2. **会员购买页**：非会员用户可见「邀请好友，免费赚会员」弱入口。
3. **首次创作完成后**：提示用户邀请好友。

### 邀请方式

- **邀请链接**：`https://aichuangzuo.com/register?ref=INVITE_CODE`
- **邀请码**：6 位字母数字，可手动填写。
- **邀请海报**：生成带二维码的海报，扫码进入注册页。

### 归因逻辑

- 新用户点击邀请链接后，`ref` 参数写入 cookie，有效期 7 天。
- 注册时若 cookie 中存在有效邀请码，则建立邀请关系。
- 若用户注册时手动填写邀请码，以手动填写为准，覆盖 cookie。
- 一个被邀请人只能绑定一个邀请人；已注册用户不能补绑。

### 分享渠道

- 复制链接
- 复制邀请码
- 保存海报图片
- 唤起微信 / QQ / 微博分享（环境支持时）

## 防刷策略

| 手段 | 说明 |
|---|---|
| 邮箱验证码 | 注册必须完成邮箱验证，限制验证码发送频率 |
| 临时邮箱黑名单 | 屏蔽一次性邮箱域名 |
| IP 频率限制 | 同一 IP 24h 内最多注册 3 个账号 |
| 设备指纹 | 同一浏览器 / 设备 7 天内只算 1 次有效邀请 |
| Cookie 标记 | 已注册设备再次注册，不计入有效邀请 |
| 邀请人上限 | 单个用户每天最多 10 个有效邀请，每月最多 50 个 |
| 奖励延迟发放 | 有效邀请判定延迟 5–15 分钟，期间风控扫描 |
| 购买返利校验 | 好友购买会员时，校验支付账号与注册信息的一致性 |

说明：纯邮箱注册无法 100% 阻止多账号，但上述手段可将批量刷量成本提到较高水平。后续用户基数扩大后，可进一步提升门槛（例如要求完成首次创作才算有效邀请）。

## 创作币与提现

### 创作币账户

每个用户拥有一个创作币账户，记录：

- 可用余额
- 累计收入
- 累计提现

### 创作币流水类型

| 类型 | 方向 | 触发条件 |
|---|---|---|
| 新用户注册奖励 | +5 | 被邀请人完成邮箱验证 |
| 首次购买返利 | +10% 订单金额（向上取整） | 被邀请人首次购买会员 |
| 续费 / 复购返利 | +5% 订单金额（向上取整） | 被邀请人后续购买会员 |
| 会员购买抵扣 | -N | 下单时使用创作币抵扣 |
| 提现 | -N | 提现申请审核通过 |

### 创作币使用场景

1. **抵扣会员购买**：1 创作币 = 1 元，无最低使用门槛，可与现金混合支付。
2. **提现**：余额 ≥ 100 创作币时，可申请支付宝转账提现。

### 提现流程

1. 用户在「邀请有礼」页面点击「提现」。
2. 系统校验余额 ≥ 100 创作币。
3. 用户填写提现金额（100 ≤ 金额 ≤ 余额）、支付宝账号、真实姓名。
4. 提交申请，状态为「待审核」。
5. 运营后台人工转账后，标记为「已完成」，系统扣减对应创作币。
6. 若审核不通过，填写原因后退回余额。

### 提现限制

- 单次提现金额 ≥ 100 创作币，且为整数。
- 每日最多提交 1 次提现申请。
- 提现申请 7 天内处理，超期未处理自动退回余额并通知用户。

## 页面与组件

### 邀请有礼主页面

#### PC 端

- **统计卡片**：已邀请人数、获得会员天数、创作币余额。
- **邀请方式**：邀请码、邀请链接、保存海报。
- **阶梯奖励进度**：可视化展示 3 人 / 5 人 / 5+ 人奖励进度。
- **好友贡献榜**：展示被邀请人状态（注册 / 已购买）及带来的创作币收益。
- **提现入口**：显示余额与提现按钮，不足 100 时提示差值。

#### 移动端

- 顶部大卡片：邀请码 + 一键复制 + 保存海报。
- 中部横向滑动统计：已邀请 / 会员天数 / 创作币。
- 下部：阶梯进度 + 好友列表 + 提现入口。

### 分享弹窗

- 默认邀请文案 + 用户可编辑的个性化邀请语。
- 按钮：复制链接、复制邀请码、保存海报、分享到微信。

### 提现弹窗

- 显示可提现余额。
- 输入提现金额、支付宝账号、真实姓名。
- 提交后提示「申请已提交，预计 7 天内到账」。

### 注册页（被邀请人侧）

- 若 URL 携带 `ref` 参数，页面顶部显示：
  > 「你收到了好友的邀请，注册并完成邮箱验证后可获得 5 创作币。」
- 注册成功后自动发放 5 创作币。

### 会员购买页

- 下单时增加「创作币抵扣」选项。
- 显示可用创作币数量，输入抵扣金额。
- 实付金额 = 订单金额 - 抵扣创作币。

## 数据结构与后端模块

### 核心数据表

| 表名 | 作用 |
|---|---|
| `user_invite_relation` | 邀请关系：邀请人、被邀请人、邀请码、来源、生效时间 |
| `invite_reward_record` | 奖励发放记录：奖励类型、数量、状态 |
| `creation_coin_account` | 创作币账户：余额、累计收入、累计提现 |
| `creation_coin_transaction` | 创作币流水：收入 / 支出类型、关联订单 / 用户、状态 |
| `withdrawal_request` | 提现申请：金额、支付宝账号、审核状态、运营备注 |
| `invite_blacklist` | 临时邮箱域名、风险 IP / 设备指纹黑名单 |

### 关键后端模块

1. **邀请服务**：生成邀请码 / 链接、注册归因、建立邀请关系。
2. **奖励服务**：阶梯会员奖励计算、创作币发放。
3. **钱包服务**：余额查询、流水记录、下单抵扣。
4. **提现服务**：提现申请、审核、打款、余额扣减。
5. **风控服务**：临时邮箱检测、IP / 设备指纹校验、邀请上限校验。

### 核心时序

```
被邀请人点击链接 → 记录 ref cookie
        ↓
被邀请人注册并验证邮箱 → 建立邀请关系
        ↓
风控校验通过 → 标记为有效邀请
        ↓
发放会员天数给邀请人 + 5 创作币给被邀请人
        ↓
被邀请人购买会员 → 生成订单 → 邀请人获得创作币返利
```

## 边界情况与风险

| 场景 | 处理方案 |
|---|---|
| 被邀请人已注册 | 不再建立邀请关系，不发放奖励 |
| 邀请码被多人使用 | 每个新用户可分别绑定，正常累计 |
| 用户删除账号 | 邀请关系保留，已发放奖励不追回 |
| 被邀请人退款 | 虚拟产品不支持退款，创作币不扣回 |
| 邀请人账号被封禁 | 冻结未提现余额，已发放会员天数不追回 |
| 运营延迟打款 | 7 天未处理自动退回余额并通知用户 |

## 验证要点

1. 邀请链接 / 邀请码能正确带入注册页，注册后建立邀请关系。
2. 邮箱验证通过后，被邀请人获得 5 创作币，邀请人累计有效邀请数 +1。
3. 达到 3 / 5 / 5+ 人时，邀请人专业版会员有效期正确延长。
4. 被邀请人购买会员后，邀请人创作币余额按 10% / 5% 正确增加。
5. 下单时创作币抵扣金额正确，实付金额 = 订单金额 - 抵扣金额。
6. 提现申请满 100 创作币才能提交，后台审核后余额正确扣减。
7. 风控规则能拦截临时邮箱、同 IP 批量注册等异常行为。
