**日期**: 2026-06-25  
**状态**: 待实现  
**关联原型**: full-prototype-v20.html  

---

## 1. 功能概述

将导出模板库从现有 12 个模板扩充至 **30 个模板**，并在弹窗顶部显示模板总数。模板按平台/场景分组，通过顶部平台标签过滤，方便用户快速定位。

## 3. UI 调整

### 3.1 弹窗头部

- 标题保持「导出模板库」。
- 标题右侧或副标题位置显示「共 30 个模板」。
- 新增平台标签栏，位于标题下方、两列主体上方。

### 3.2 平台标签栏

- 标签项：全部、公众号、小红书、今日头条、百家号、知乎、抖音图文、通用风格。
- 默认选中「全部」，展示全部 30 个模板。
- 切换标签时，右侧列表只显示该平台模板，左侧大预览自动切换为该标签下第一个模板。
- 标签栏支持横向滚动（移动端/小屏适配）。

### 3.3 右侧模板列表

- 保持现有卡片式行布局。
- 模板行左侧保留平台色块图标（与平台主色一致）。
- 选中态、hover 态保持现有交互。

## 4. 数据结构调整

### 4.1 `templatePresets`

每个模板对象新增 `platform` 字段，用于分组和过滤：

```javascript
{ key: 'wechat', name: '公众号标准模板', platform: 'wechat', ... }
```

平台 key 统一为：
- `wechat`：公众号
- `xiaohongshu`：小红书
- `toutiao`：今日头条
- `baijiahao`：百家号
- `zhihu`：知乎
- `douyin`：抖音图文
- `general`：通用风格

### 4.2 `templateLargeStyles`

为新增模板补充大预览样式定义，保证左侧实时预览能正确渲染。

### 4.3 `templateStyles` / `templateHeadingTexts`

预览页（screen-preview）直接点击模板卡片时，新增模板同样需要能应用对应样式。

## 5. 技术要点

- 纯前端 HTML/CSS/JS 原型实现，无后端依赖。
- 弹窗渲染时根据 `platform` 字段动态生成标签和过滤列表。
- 标签切换后，自动更新左侧大预览为当前标签下第一个模板，并更新「应用此模板」按钮状态。
- 从模板库应用模板后，继续同步更新 `screen-preview` 主预览区（沿用现有 `applyTemplateFeedback` 逻辑）。
- 新增模板的大预览和卡片小图风格需与平台调性保持一致。

## 6. 涉及原型页面

- `openTemplateLibrary` 弹窗组件（主要改动）。
- `screen-preview` 预览页（右侧模板列表、模板卡片样式）。
