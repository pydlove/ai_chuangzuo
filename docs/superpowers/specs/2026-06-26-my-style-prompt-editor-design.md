# 我的风格提示词编辑器设计

**日期**: 2026-06-26  
**状态**: 已确认，待实现  
**关联文件**: `shared.js`、`shared.css`、`create.html` 等使用 `openStyleLibrary()` 的页面  

---

## 1. 功能概述

在「风格库 → 我的风格」tab 中，点击「新建我的风格」或已有风格的「编辑提示词」时，不再弹出 `alert`，而是打开一个可填写/修改风格名称与提示词的编辑器。保存后即时反映在「我的风格」列表中。

## 2. 设计决策

- **编辑器形态**: 在风格库弹窗的内容区内切换为子视图，不叠加新弹窗，空间充足且导航清晰。
- **数据存储**: 本次仅在前端内存中维护，刷新页面后数据丢失，符合原型阶段需求。
- **字段范围**: 仅向用户暴露「风格名称」和「风格提示词」两个字段；`desc`、`count` 等字段沿用默认值。
- **编辑范围**: 新建与编辑共用同一套表单组件。

## 3. 数据模型

用户风格对象沿用现有结构：

```js
{
  name: '风格名称',   // 必填，最多 20 字
  desc: '',           // 创建时不填写，展示时省略
  count: 0,           // 新建时默认为 0
  prompt: '提示词'    // 必填，最多 1000 字
}
```

## 4. 交互流程

1. 用户点击「新建我的风格」或已有风格的「编辑提示词」。
2. 风格库弹窗的内容区切换为编辑子视图：
   - 顶部显示「← 返回」按钮 + 标题（新建我的风格 / 编辑提示词）。
   - 中部为表单。
   - 底部固定「保存」「取消」按钮。
3. 用户填写或修改后点击「保存」。
4. 校验通过后回到「我的风格」列表：
   - 新建：新卡片插入列表最前面。
   - 编辑：被编辑的卡片原地更新。
5. 点击「取消」或「返回」直接回到列表，不保留改动。

## 5. 表单字段

| 字段 | 类型 | 必填 | 限制 | 说明 |
|---|---|---|---|---|
| 风格名称 | 单行输入 | 是 | 最多 20 字 | 作为卡片标题展示 |
| 风格提示词 | 多行文本域 | 是 | 最多 1000 字 | 实时显示字数统计 `0/1000` |

## 6. 校验与错误处理

| 场景 | 表现 |
|---|---|
| 名称为空 | 输入框边框变红，下方显示「请输入风格名称」 |
| 提示词为空 | 文本域边框变红，下方显示「请输入风格提示词」 |
| 提示词超过 1000 字 | 字数统计变红，保存按钮禁用 |
| 校验未通过时点击保存 | 不执行保存，聚焦到第一个错误字段 |

## 7. 视觉与样式

- 子视图沿用风格库弹窗的白色卡片背景与圆角。
- 表单字段间距、字号、边框颜色与现有输入框保持一致。
- 保存按钮使用主色 `#07c160`，未通过校验时置灰。
- 返回按钮为文字链形式，位于子视图左上角。

## 8. 实现位置

- **`shared.js`**: 在 `openStyleLibrary()` 内新增 `renderStyleEditor(mode, index)` 函数，处理新建/编辑的渲染、校验与保存逻辑；修改「新建我的风格」和「编辑提示词」的点击事件。
- **`shared.css`**: 补充编辑器子视图、错误态、字数统计等少量样式。
- **各页面 HTML**: 无需改动，`create.html` 等页面直接复用更新后的 `openStyleLibrary()`。

## 9. 边界情况

- 新建风格保存后，默认未被选中；如需应用，用户需手动点击卡片选择。
- 编辑过程中切换 tab（系统预设 / 我的风格）应视为取消编辑，回到列表。
- 弹窗关闭后再打开，编辑状态不保留。

## 10. 测试要点

- 新建风格后列表中立即出现且字段正确。
- 编辑已有风格后卡片内容同步更新。
- 名称为空、提示词为空、提示词超长时保存被阻止并给出提示。
- 取消/返回不修改列表数据。
- 在 PC 与移动端视口下表单均可正常显示。
